[1886. Determine Whether Matrix Can Be Obtained By Rotation](https://leetcode.com/problems/determine-whether-matrix-can-be-obtained-by-rotation/)

# Intuition

Rotation options are limited:

```
0°   (original)
90°
180°
270°
```

If any rotation of `mat` equals `target`, return true.

Matrix rotation rule (90° clockwise):

```
new[i][j] = mat[n-1-j][i]
```

---

# Approach

Two valid strategies:
### Strategy 1 (In-place simulation)

Rotate matrix step-by-step and compare.

### Strategy 2 (Direct comparison without modifying matrix)

Check all 4 rotations mathematically.

Use mapping:

```
0°   → mat[i][j]
90°  → mat[n-1-j][i]
180° → mat[n-1-i][n-1-j]
270° → mat[j][n-1-i]
```

Compare with:

```
target[i][j]
```

## Algorithm

For each rotation:

```
assume match = true

for all i,j:
    if rotated value != target[i][j]:
        match = false
        break

if match → return true
```

If none match:

```
return false
```

---

# Complexity

- **Time Complexity:** $O(n²)$
	4 rotations × full matrix scan.

- **Space Complexity:** $O(1)$
	No extra matrix used.

---

# Code

```typescript 
function findRotation(mat: number[][], target: number[][]): boolean {

    const n = mat.length;

    const check = (type: number): boolean => {

        for (let i = 0; i < n; i++) {
            for (let j = 0; j < n; j++) {

                let val;

                if (type === 0) val = mat[i][j];
                else if (type === 1) val = mat[n - 1 - j][i];
                else if (type === 2) val = mat[n - 1 - i][n - 1 - j];
                else val = mat[j][n - 1 - i];

                if (val !== target[i][j]) return false;
            }
        }

        return true;
    };

    for (let t = 0; t < 4; t++) {
        if (check(t)) return true;
    }

    return false;
};

```
