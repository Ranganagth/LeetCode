[1009. Complement of Base 10 Integer](https://leetcode.com/problems/complement-of-base-10-integer/)

## Intuition

Binary complement flips every bit:

```
0 → 1
1 → 0
```

Example:

```
5 = 101
flip → 010 = 2
```

Flipping must occur **only within the significant bits of `n`**.

Example:

```
5 = 000000101   (32-bit representation)
```

Flipping all bits gives incorrect result. Only the bits up to the **most significant bit** must be flipped.

Therefore:

1. Determine the length of the binary representation.
2. Build a mask containing `1`s in those positions.
3. XOR the mask with `n`.

Property:

```
n XOR 111..111  = flipped bits
```

---

## Approach

### Step 1 - Create bit mask

For number `n`, determine the smallest number containing only `1`s that fully covers its bits.

Example:

```
n = 5  (101)

mask = 111
```

Method:

```
mask = (1 << bitLength) - 1
```

Example:

```
bitLength = 3
mask = (1 << 3) - 1
     = 8 - 1
     = 7 (111)
```

### Step 2 - XOR with mask

```
result = mask ^ n
```

Example:

```
mask = 111
n    = 101

111
101
---
010 = 2
```

## Algorithm

1. If `n == 0`, return `1`.
2. Find bit length of `n`.
3. Construct mask `(1 << bitLength) - 1`.
4. Return `mask ^ n`.

---

# Complexity

- **Time Complexity:**
```
O(1)
```

Bit operations occur on fixed-size integers.

- **Space Complexity:**
```
O(1)
```

No additional memory structures used.

---

# Code

```typescript
function bitwiseComplement(n: number): number {

    if (n === 0) return 1;

    let bitLength = Math.floor(Math.log2(n)) + 1;

    let mask = (1 << bitLength) - 1;

    return mask ^ n;
};

```
