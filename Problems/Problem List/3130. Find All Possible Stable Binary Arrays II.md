[3130. Find All Possible Stable Binary Arrays II](https://leetcode.com/problems/find-all-possible-stable-binary-arrays-ii/)

### Core Constraint

Condition *“every subarray with size > limit must contain both 0 and 1”* implies:

No more than **limit consecutive identical bits** can appear.

Therefore the array is a sequence of **blocks** of 0s and 1s where each block length ≤ `limit`.

Total counts must be exactly:

* `zero` zeros
* `one` ones

---

# Intuition

Construct the binary array incrementally.

At any step:

* Some zeros and ones remain unused.
* The previous placed bit determines the current block type.
* A block cannot exceed length `limit`.

Instead of tracking explicit block length, use a **count subtraction trick**.

Observation:

If placing a `0`:

* All sequences ending with `0` or `1` before it are valid.
* But sequences that already formed `limit + 1` zeros must be removed.

This converts a potentially **O(limit)** transition into **O(1)**.

Define state:

```
dp[z][o][last]
```

Where:

* `z` zeros remaining
* `o` ones remaining
* `last` = last placed bit (`0` or `1`)

Meaning:

Number of stable arrays that can be formed.

---

# DP Transition

### When last bit = 0

Place another `0`.

```
dp(z,o,0) =
    dp(z-1,o,0)   // continue zero block
  + dp(z-1,o,1)   // start new zero block
```

But this allows blocks longer than `limit`.

Remove invalid cases where `limit+1` zeros occur:

```
if z > limit
    subtract dp(z-limit-1, o, 1)
```

Reason:

Those sequences previously ended with `1` then added `limit+1` zeros.

### When last bit = 1

Symmetric case.

```
dp(z,o,1) =
    dp(z,o-1,0)
  + dp(z,o-1,1)
```

Remove sequences forming `limit+1` ones.

```
if o > limit
    subtract dp(z, o-limit-1, 0)
```

# Base Conditions

If only one type remains:

### Only ones left

Valid only if:

```
lastBit == 1
one <= limit
```

Otherwise invalid.

### Only zeros left

Valid only if:

```
lastBit == 0
zero <= limit
```

---

# Algorithm

1. Use recursion + memoization.
2. Track `(zeros remaining, ones remaining, last bit)`
3. Use subtraction trick to enforce limit.
4. Final result = sequences ending with `0` + sequences ending with `1`.

---
# Complexity

Let:

```
Z = zero
O = one
```

- **Time Complexity:** $O(Z * O)$
	Each state computed once.

Transitions are constant time due to subtraction optimization.

- **Space Complexity:** $O(Z * O)$
	Memoization table storing two states per cell.

---
# Code

```ts
const MOD = 1000000007;

function numberOfStableArrays(zero: number, one: number, limit: number): number {

    const memo: number[][][] = Array.from({ length: zero + 1 }, () =>
        Array.from({ length: one + 1 }, () => [-1, -1])
    );

    function dp(z: number, o: number, last: number): number {

        if (z === 0) {
            if (last === 0 || o > limit) return 0;
            return 1;
        }

        if (o === 0) {
            if (last === 1 || z > limit) return 0;
            return 1;
        }

        if (memo[z][o][last] !== -1) return memo[z][o][last];

        let res = 0;

        if (last === 0) {

            res = (dp(z - 1, o, 0) + dp(z - 1, o, 1)) % MOD;

            if (z > limit) {
                res = (res - dp(z - limit - 1, o, 1) + MOD) % MOD;
            }

        } else {

            res = (dp(z, o - 1, 0) + dp(z, o - 1, 1)) % MOD;

            if (o > limit) {
                res = (res - dp(z, o - limit - 1, 0) + MOD) % MOD;
            }
        }

        memo[z][o][last] = res;
        return res;
    }

    return (dp(zero, one, 0) + dp(zero, one, 1)) % MOD;
};

```

# Example Walkthrough

### Example 1

```
zero = 1
one = 2
limit = 1
```

Constraint: no consecutive identical bits allowed.

Possible permutations of `{0,1,1}`:

```
011 ❌ (two consecutive 1s)
101 ✅
110 ❌
```

Only valid array:

```
[1,0,1]
```

Result:

```
1
```

### Example 2

```
zero = 3
one = 3
limit = 2
```

Constraint:

```
max 2 consecutive 0s
max 2 consecutive 1s
```

Valid sequences include:

```
001011
001101
010011
010101
...
```

Total:

```
14
```

DP counts all such combinations while removing sequences exceeding block size.

# Why This Is Optimal

Brute force generation:

```
O(2^(zero+one))
```

Tracking run length DP:

```
O(Z * O * limit)
```

Optimized DP with prefix subtraction:

```
O(Z * O)
```

Necessary because constraints allow:

```
zero, one ≤ 1000
```

Naive DP would exceed time limits.
