[1833. Maximum Ice Cream Bars](https://leetcode.com/problems/maximum-ice-cream-bars/)

# Intuition

Since the problem requires solving via counting sort, the key idea is to:
- Count the frequency of each cost.
- Iterate over costs in ascending order, buying as many ice cream bars as possible starting from the cheapest.
- Keep track of how many bars are bought until coins run out or all bars are purchased.

This approach ensures we buy the maximum number of bars with the available coins, prioritizing cheaper ones first.

---

# Approach

1. Find the maximum cost in `costs`.
2. Create a frequency array `freq` of size `maxCost + 1` to count how many bars have each cost.
3. Iterate through the `freq` array from the lowest cost to the highest:
   - Determine how many bars of this cost can be bought (`canBuy`), which is the minimum of available bars and how many can be bought with remaining coins.
   - Update the total count of bars bought.
   - Deduct the spent coins.
   - If coins are insufficient to buy more of the current cost, break early.
4. Return the total number of bars bought.

---

# Complexity
- **Time complexity:**  $$O(n + M)$$ where `n` is the length of `costs`, and `M` is the maximum cost value (up to $10^5$). The counting sort step is O(n), and the iteration over costs is O(M).
  
- **Space complexity:**  $$O(M)$$ for the frequency array.

---

# Code

```typescript
function maxIceCream(costs: number[], coins: number): number {
    const maxCost = Math.max(...costs);
    const freq: number[] = Array(maxCost + 1).fill(0);
    for (const cost of costs) {
        freq[cost]++;
    }

    let res = 0;
    for (let cost = 1; cost <= maxCost; cost++) {
        if (freq[cost] === 0) continue;
        const canBuy = Math.min(freq[cost], Math.floor(coins / cost));
        res += canBuy;
        coins -= canBuy * cost;
        if (coins < cost) break;
    }
    return res;
};

```

# Example walkthrough
## Example 1:
`costs = [1,3,2,4,1], coins = 7`

- Count frequencies:
  - 1: 2
  - 2: 1
  - 3: 1
  - 4: 1
- Buy:
  - cost=1: min(2, 7/1=7) = 2 → buy 2, coins=7-2=5
  - cost=2: min(1, 5/2=2) = 1 → buy 1, coins=5-2=3
  - cost=3: min(1, 3/3=1) = 1 → buy 1, coins=3-3=0
  - cost=4: can't buy (coins=0)
- Total bought = 2+1+1=4

**Output:** 4

## Example 2:
`costs = [10,6,8,7,7,8], coins=5`

- Cheapest cost is 6, but 6 > 5 coins, so can't buy any.
- Output: 0

## Example 3:
`costs = [1,6,3,1,2,5], coins=20`

- Frequencies:
  - 1: 2
  - 2: 1
  - 3: 1
  - 5: 1
  - 6: 1
- Buy:
  - cost=1: min(2, 20/1=20)=2 → buy 2, coins=20-2=18
  - cost=2: min(1, 18/2=9)=1 → buy 1, coins=18-2=16
  - cost=3: min(1, 16/3=5)=1 → buy 1, coins=16-3=13
  - cost=5: min(1, 13/5=2)=1 → buy 1, coins=13-5=8
  - cost=6: min(1, 8/6=1)=1 → buy 1, coins=8-6=2
- Total bought = 2+1+1+1+1=6

**Output:** 6
