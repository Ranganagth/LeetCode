[1727. Largest Submatrix With Rearrangements](https://leetcode.com/problems/largest-submatrix-with-rearrangements/)

# Intuition

Reordering columns means:

```
column positions do not matter
only heights of consecutive 1s matter
```

Convert the problem into a histogram per row.

For each cell:

```
height[i][j] = number of consecutive 1s ending at (i,j)
```

Example:

```
matrix:
1 0 1
1 1 1

heights:
1 0 1
2 1 2
```

Now, for each row:

* You can rearrange columns freely.
* So sort heights in descending order.
* Try forming a rectangle using first `k` columns.

Area:

```
area = height[k] * (k + 1)
```

This works because:

```
minimum height among chosen columns determines rectangle height
```

---

# Approach

### Step 1 - Build heights

```
if matrix[i][j] == 1:
    height[i][j] = height[i-1][j] + 1
else:
    height[i][j] = 0
```

### Step 2 - For each row

1. Copy heights
2. Sort descending
3. Compute max area

```
for k in 0..n-1:
    area = heights[k] * (k+1)
```

### Step 3 - Track global maximum

## Algorithm

For each row:

```
build histogram
sort descending
compute max rectangle
```

---

# Complexity

Let:

```
m = rows
n = columns
```

- **Time Complexity:**
	Building heights: $O(m * n)$
	Sorting each row: $O(m * n log n)$
	**Total:** $O(m * n log n)$

- **Space Complexity:** $O(m * n)$
	*Can be optimized to:* $O(n)$ by reusing a single array.

# Code

```typescript
function largestSubmatrix(matrix: number[][]): number {

    const m = matrix.length;
    const n = matrix[0].length;

    const height = Array.from({ length: m }, () => new Array(n).fill(0));

    for (let j = 0; j < n; j++) {
        height[0][j] = matrix[0][j];
    }

    for (let i = 1; i < m; i++) {
        for (let j = 0; j < n; j++) {
            if (matrix[i][j] === 1) {
                height[i][j] = height[i - 1][j] + 1;
            } else {
                height[i][j] = 0;
            }
        }
    }

    let maxArea = 0;

    for (let i = 0; i < m; i++) {

        const row = [...height[i]];

        row.sort((a, b) => b - a);

        for (let j = 0; j < n; j++) {

            const h = row[j];
            const width = j + 1;

            maxArea = Math.max(maxArea, h * width);
        }
    }

    return maxArea;
};

```

## Key Insight

- Column order is irrelevant
- Only heights matter
- Sorting simulates optimal column rearrangement
