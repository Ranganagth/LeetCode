[1594. Maximum Non Negative Product in a Matrix](https://leetcode.com/problems/maximum-non-negative-product-in-a-matrix/)

# Intuition

Products with negatives behave non-monotonically:

```
positive * negative → negative
negative * negative → positive
```

Tracking only maximum fails because:

```
a very small (negative) value can become largest positive later
```

So each cell must track:

```
max product reaching here
min product reaching here
```

---

# Core Idea

Dynamic Programming with two states:

```
maxDP[i][j] → maximum product ending at (i,j)
minDP[i][j] → minimum product ending at (i,j)
```

Transitions depend on sign of current cell.

## Transition

From top `(i-1,j)` and left `(i,j-1)`.

Let:

```
val = grid[i][j]
```

If `val >= 0`:

```
max = max(prevMax) * val
min = min(prevMin) * val
```

If `val < 0`:

```
max = min(prevMin) * val
min = max(prevMax) * val
```

Take both directions into account.

## Initialization

Start:

```
maxDP[0][0] = grid[0][0]
minDP[0][0] = grid[0][0]
```

First row and column handled separately.

## Final Answer

```
result = maxDP[m-1][n-1]
```

If:

```
result < 0 → return -1
else → result % MOD
```

---

# Complexity

- **Time Complexity:** $O(m * n)$
	Each cell processed once.

- **Space Complexity:** $O(m * n)$
	**Can be optimized to:** $O(n)$
	using rolling arrays.

---

# Code

```typescript 
function maxProductPath(grid: number[][]): number {

    const MOD = 1e9 + 7;
    const m = grid.length;
    const n = grid[0].length;

    const maxDP: number[][] = Array.from({ length: m }, () => new Array(n).fill(0));
    const minDP: number[][] = Array.from({ length: m }, () => new Array(n).fill(0));

    maxDP[0][0] = grid[0][0];
    minDP[0][0] = grid[0][0];

    // first row
    for (let j = 1; j < n; j++) {
        maxDP[0][j] = maxDP[0][j - 1] * grid[0][j];
        minDP[0][j] = maxDP[0][j];
    }

    // first column
    for (let i = 1; i < m; i++) {
        maxDP[i][0] = maxDP[i - 1][0] * grid[i][0];
        minDP[i][0] = maxDP[i][0];
    }

    for (let i = 1; i < m; i++) {
        for (let j = 1; j < n; j++) {

            const val = grid[i][j];

            const candidates = [
                maxDP[i - 1][j] * val,
                minDP[i - 1][j] * val,
                maxDP[i][j - 1] * val,
                minDP[i][j - 1] * val
            ];

            maxDP[i][j] = Math.max(...candidates);
            minDP[i][j] = Math.min(...candidates);
        }
    }

    const result = maxDP[m - 1][n - 1];

    if (result < 0) return -1;

    return result % MOD;
};

```

# Example Walkthrough

### Example 1

```
grid =
[
[1,-2,1],
[1,-2,1],
[3,-4,1]
]
```

### Step-by-step

Start:

```
(0,0) = 1
```

Row 0:

```
1 → -2 → -2
```

Column 0:

```
1 → 1 → 3
```

At (1,1) = -2

From top:

```
-2 * -2 = 4
```

From left:

```
1 * -2 = -2
```

Track:

```
max = 4
min = -2
```

At (2,1) = -4

From candidates:

```
max = (-2 * -4) = 8
min = (4 * -4) = -16
```

At (2,2) = 1

```
max = 8
```

### Result

```
8
```


### Example 2

```
grid =
[
[1,3],
[0,-4]
]
```

Path:

```
1 → 0 → -4 = 0
```

Zero dominates:

```
result = 0
```

## Key Insight

Track both min and max because negatives can flip outcomes
