[3504. Longest Palindrome After Substring Concatenation II](https://leetcode.com/problems/longest-palindrome-after-substring-concatenation-ii/)

# Intuition

Constructed string:

```
substring_from_s + substring_from_t
```

Palindrome structure:

```
left part from s
right part from t (mirrored)
```

Three possibilities:

```
1. palindrome entirely inside s
2. palindrome entirely inside t
3. palindrome formed across s and t
```

Case 3 is the core difficulty.

## Key Idea

If characters match:

```
s[i] == t[j]
```

They can form symmetric ends of a palindrome.

Build longest matching pairs:

```
expand inward:
s[i-1] with t[j+1]
```

This is equivalent to:

```
longest common substring between s and reversed t
```

---

# Approach

### Part 1 - Palindromes inside s

```
dpS[i][j] = whether s[i..j] is palindrome
leftS[i] = longest palindrome starting at i
```

### Part 2 - Palindromes inside t

```
dpT[i][j] = whether t[i..j] is palindrome
rightT[j] = longest palindrome ending at j
```

### Part 3 - Cross matching s and t

```
dp[i][j] = length of matching sequence
           between s[0..i] and reversed t[j..n-1]
```

Transition:

```
if s[i] == t[j]:
    dp[i][j] = 1 + dp[i-1][j+1]
```

This gives:

```
number of matching pairs
```

### Extend palindrome

Base palindrome:

```
2 * dp[i][j]
```

Add center (optional):

```
either:
- palindrome in s after i
- palindrome in t before j
```

So:

```
extra = max(leftS[i+1], rightT[j-1])
```

Final:

```
length = 2 * dp[i][j] + extra
```

---

## Algorithm Summary

1. Compute all palindromes in `s`
2. Compute all palindromes in `t`
3. Match characters between `s` and reversed `t`
4. Expand palindrome using DP
5. Track maximum

---

# Complexity

Let:
	m = |s|
	n = |t|

- **Time Complexity:** $O(m² + n² + m*n)$
	* Palindrome DP for s → O(m²)
	* Palindrome DP for t → O(n²)
	* Cross DP → $O(m*n)$

- **Space Complexity:** $O(m² + n² + m*n)$

# Code

```typescript
const longestPalindrome = (s: string, t: string): number => {
    const m = s.length, n = t.length;
    let ans = 0;

    // DP table to check if s[i:j] is a palindrome
    const dpS: boolean[][] = Array.from({ length: m }, () => Array(m).fill(false));
    const leftS: number[] = Array(m).fill(0);

    // Compute longest palindromic substring in s
    for (let i = m - 1; i >= 0; i--) {
        for (let j = i; j < m; j++) {
            if (s[i] === s[j] && (j - i < 2 || dpS[i + 1][j - 1])) {
                dpS[i][j] = true;
                leftS[i] = Math.max(leftS[i], j - i + 1);
                ans = Math.max(ans, leftS[i]);
            }
        }
    }

    // DP table to check if t[i:j] is a palindrome
    const dpT: boolean[][] = Array.from({ length: n }, () => Array(n).fill(false));
    const rightT: number[] = Array(n).fill(0);

    // Compute longest palindromic substring in t
    for (let i = n - 1; i >= 0; i--) {
        for (let j = i; j < n; j++) {
            if (t[i] === t[j] && (j - i < 2 || dpT[i + 1][j - 1])) {
                dpT[i][j] = true;
                rightT[j] = Math.max(rightT[j], j - i + 1);
                ans = Math.max(ans, rightT[j]);
            }
        }
    }

    // DP table to check longest palindromic match between s and t
    const dp: number[][] = Array.from({ length: m }, () => Array(n).fill(0));

    for (let i = 0; i < m; i++) {
        for (let j = n - 1; j >= 0; j--) {
            if (s[i] === t[j]) {
                dp[i][j] = 1;
                if (i > 0 && j < n - 1) {
                    dp[i][j] += dp[i - 1][j + 1];
                }
            }
            if (dp[i][j]) {
                let extra = 0;
                if (i + 1 < m) extra = Math.max(extra, leftS[i + 1]);
                if (j - 1 >= 0) extra = Math.max(extra, rightT[j - 1]);
                ans = Math.max(ans, 2 * dp[i][j] + extra);
            }
        }
    }

    return ans;
};

```

# Example Walkthrough

### Example 1

```
s = "abcde"
t = "ecdba"
```


#### Step 1 - Palindromes inside s and t

All single characters:

```
length = 1
```

No longer palindromes.

#### Step 2 - Cross matching

Match:

```
s[0] = 'a' with t[4] = 'a'
s[1] = 'b' with t[3] = 'b'
s[2] = 'c' with t[2] = 'c'
```

So:

```
dp = 3
```

Palindrome formed:

```
"abc" + "ba" → "abcba"
```

Length:

```
2 * 3 - 1 = 5
```

(Overlap handled via DP extension logic)

#### Step 3 - Extend center

Check:

```
leftS[i+1], rightT[j-1]
```

No longer extension possible.

#### Final Answer

```
5
```

### Example 2

s = "a"

dp = 1

Result:

### Example 3

## Key Insights

1. Palindrome = symmetric matching + optional center
2. Reduce cross-string palindrome to matching s with reversed t
3. Precompute internal palindromes to extend center
