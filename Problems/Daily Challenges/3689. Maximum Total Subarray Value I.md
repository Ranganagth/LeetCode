[3689. Maximum Total Subarray Value I](https://leetcode.com/problems/maximum-total-subarray-value-i/)

# Intuition

The key insight is that **we can choose the same subarray multiple times**, and we want to maximize the total sum of `(max - min)` over `k` selections.

The maximum possible value for any single subarray is `max(nums) - min(nums)`, which is achieved by any subarray that contains both the global maximum and global minimum elements.

Since we can:
1. Choose any subarray (including overlapping ones)
2. Choose the same subarray multiple times
3. Want to maximize the total sum

The optimal strategy is to **always choose a subarray with the maximum possible value** `(max(nums) - min(nums))` exactly `k` times.

---

# Approach

1. Find the **global maximum** and **global minimum** of the array.
2. The maximum value any subarray can have is `max(nums) - min(nums)`.
3. Since we can choose the same subarray `k` times, the maximum total value is:
   $$
   (max(nums) - min(nums)) × k
$$ 
---

# Complexity

- **Time complexity:** 
  $$O(n)$$ to find the max and min.
- **Space complexity:** 
  $$O(1)$$

---

# Code

```typescript
function maxTotalValue(nums: number[], k: number): number {
    let minVal = Infinity;
    let maxVal = -Infinity;
    
    for (const x of nums) {
        minVal = Math.min(minVal, x);
        maxVal = Math.max(maxVal, x);
    }
    
    return (maxVal - minVal) * k;
};

```

# Example walkthrough

## Example 1:
`nums = [1,3,2]`, `k = 2`

- `min(nums) = 1`
- `max(nums) = 3`
- Maximum subarray value = `3 - 1 = 2`
- We can choose any subarray containing both 1 and 3 (e.g., `[1,3]` or `[1,3,2]`) twice
- Total = `2 × 2 = 4` 

## Example 2:
`nums = [4,2,5,1]`, `k = 3`

- `min(nums) = 1`
- `max(nums) = 5`
- Maximum subarray value = `5 - 1 = 4`
- We can choose any subarray containing both 1 and 5 (e.g., `[4,2,5,1]` or `[5,1]`) three times
- Total = `4 × 3 = 12` 

---

# Why this works

- Any subarray that contains both the global max and global min will have value `max(nums) - min(nums)`.
- No subarray can have a value greater than this.
- Since we can choose the same subarray multiple times, we simply pick the best one `k` times.
- This greedy approach is optimal because we're maximizing each of the `k` selections independently.
