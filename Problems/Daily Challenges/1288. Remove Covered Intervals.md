[1288. Remove Covered Intervals](https://leetcode.com/problems/remove-covered-intervals/)

# Intuition

To efficiently count non-covered intervals, we can use a **greedy sorting approach**:

1. **Sort intervals** by:
   - Start point in ascending order (primary).
   - End point in descending order (secondary) - this ensures that if two intervals have the same start, the one with the larger end comes first.

2. **Greedy traversal**:
   - After sorting, iterate through intervals.
   - Track the maximum end point seen so far.
   - If the current interval's end is <= max end seen, it's covered by a previous interval.
   - Otherwise, it's not covered, so increment the count and update max end.

**Why this works:**
- After sorting by start (ascending) and end (descending), if interval A comes before interval B:
  - If A.start == B.start, then A.end >= B.end (due to secondary sort), so B is covered by A.
  - If A.start < B.start, then B is covered by A only if B.end <= A.end.
- By tracking the maximum end point, we can determine coverage in a single pass.

---

# Approach

1. Sort intervals by start point (ascending), then by end point (descending).
2. Initialize `count = 1` and `maxEnd = intervals[0][1]`.
3. Iterate from index 1 to the end:
   - If `intervals[i][1] <= maxEnd`, the interval is covered, skip it.
   - Otherwise, increment count and update `maxEnd`.
4. Return count.

---

# Complexity

- **Time complexity:** $$O(n \log n)$$ for sorting, where `n` is the number of intervals.

- **Space complexity:**  $$O(1)$$ if we don't count the space used by sorting (or O(n) if we do).

---

# Code

```typescript
function removeCoveredIntervals(intervals: number[][]): number {
    // Sort by start (ascending), then by end (descending)
    intervals.sort((a, b) => {
        if (a[0] !== b[0]) {
            return a[0] - b[0];
        }
        return b[1] - a[1];
    });

    let count = 1;
    let maxEnd = intervals[0][1];

    for (let i = 1; i < intervals.length; i++) {
        // If current interval's end is greater than max end seen so far,
        // it's not covered by any previous interval
        if (intervals[i][1] > maxEnd) {
            count++;
            maxEnd = intervals[i][1];
        }
        // Otherwise, it's covered by a previous interval
    }

    return count;
};

```

# Example walkthrough
## Example 1:
**Input:** `intervals = [[1,4],[3,6],[2,8]]`

**Step 1: Sort**
- Sort by start (ascending), then end (descending):
- `[[1,4],[2,8],[3,6]]`

**Step 2: Greedy traversal**
- Start: count = 1, maxEnd = 4
- i = 1: intervals[1] = [2,8], 8 > 4 → count = 2, maxEnd = 8
- i = 2: intervals[2] = [3,6], 6 <= 8 → covered, skip

**Output:** `2` 

## Example 2:
**Input:** `intervals = [[1,4],[2,3]]`

**Step 1: Sort**
- Sort by start (ascending), then end (descending):
- `[[1,4],[2,3]]`

**Step 2: Greedy traversal**
- Start: count = 1, maxEnd = 4
- i = 1: intervals[1] = [2,3], 3 <= 4 → covered, skip

**Output:** `1`
