[3336. Find the Number of Subsequences With Equal GCD](https://leetcode.com/problems/find-the-number-of-subsequences-with-equal-gcd/description/?envType=daily-question&envId=2026-07-14)

# Intuition

This is a complex combinatorial problem involving:
- **Subsequence generation** with disjoint constraints
- **GCD computation** for subsequences
- **Dynamic Programming** to count valid pairs efficiently

The key insight is that for each element in `nums`, we have 3 choices:
1. Include it in subsequence 1
2. Include it in subsequence 2 
3. Don't include it in either

We need to track the GCD of both subsequences as we build them and count pairs where both GCDs are equal.

---

# Approach

## Strategy: Dynamic Programming with State Tracking

1. **State Definition**: `dp[i][gcd1][gcd2]` = number of ways to process first `i` elements such that:
   - Subsequence 1 has GCD = `gcd1`
   - Subsequence 2 has GCD = `gcd2`

2. **Transitions**: For each element `nums[i]`, we have 3 choices:
   - **Add to seq1**: Update GCD of seq1 → `gcd(gcd1, nums[i])`
   - **Add to seq2**: Update GCD of seq2 → `gcd(gcd2, nums[i])`
   - **Skip**: Keep both GCDs unchanged

3. **Base Case**: Start with `dp[0][0][0] = 1` (empty subsequences)

4. **Answer**: Sum all `dp[n][g][g]` where `g > 0` (both subsequences non-empty with equal GCD)

5. **Optimization**: Use a map/object to store only non-zero states (sparse DP)

---

# Complexity

- **Time complexity:** $$O(n \times M^2 \times \log M)$$
  - `n` = length of nums
  - `M` = max value in nums (200)
  - For each element, we iterate through all possible GCD pairs
  - GCD computation: $$O(\log M)$$

- **Space complexity:** $$O(M^2)$$ - storing DP states for all possible GCD pairs

# Code

```typescript
function subsequencePairCount(nums: number[]): number {
    const MOD = 1e9 + 7;
    const n = nums.length;
    
    // Helper function to compute GCD
    const gcd = (a: number, b: number): number => {
        if (b === 0) return a;
        return gcd(b, a % b);
    };
    
    // dp[gcd1][gcd2] = count of ways to have seq1 with gcd1 and seq2 with gcd2
    // Use Map for sparse storage
    let dp = new Map<string, number>();
    dp.set("0,0", 1); // Base case: both subsequences empty
    
    // Process each element
    for (const num of nums) {
        const newDp = new Map<string, number>();
        
        for (const [key, count] of dp.entries()) {
            const [gcd1Str, gcd2Str] = key.split(',');
            const gcd1 = parseInt(gcd1Str);
            const gcd2 = parseInt(gcd2Str);
            
            // Option 1: Don't include num in either subsequence
            const skipKey = `${gcd1},${gcd2}`;
            newDp.set(skipKey, ((newDp.get(skipKey) || 0) + count) % MOD);
            
            // Option 2: Add num to subsequence 1
            const newGcd1 = gcd1 === 0 ? num : gcd(gcd1, num);
            const addToSeq1Key = `${newGcd1},${gcd2}`;
            newDp.set(addToSeq1Key, ((newDp.get(addToSeq1Key) || 0) + count) % MOD);
            
            // Option 3: Add num to subsequence 2
            const newGcd2 = gcd2 === 0 ? num : gcd(gcd2, num);
            const addToSeq2Key = `${gcd1},${newGcd2}`;
            newDp.set(addToSeq2Key, ((newDp.get(addToSeq2Key) || 0) + count) % MOD);
        }
        
        dp = newDp;
    }
    
    // Count pairs where both subsequences are non-empty and have equal GCD
    let result = 0;
    for (const [key, count] of dp.entries()) {
        const [gcd1Str, gcd2Str] = key.split(',');
        const gcd1 = parseInt(gcd1Str);
        const gcd2 = parseInt(gcd2Str);
        
        if (gcd1 > 0 && gcd1 === gcd2) {
            result = (result + count) % MOD;
        }
    }
    
    return result;
};

```

# Example Walkthrough

## Example 1:

**Input:** `nums = [1,2,3,4]`

**Explanation:**

**Key Observation**: All elements have GCD = 1 when combined

**Processing:**
- Start: `dp = {(0,0): 1}`
- After num=1: States include `(1,0)`, `(0,1)`, `(1,1)`, etc.
- After num=2: GCD(1,2) = 1, so many states collapse to GCD=1
- Continue for 3 and 4...

**Valid pairs** (both GCD = 1):
- Multiple combinations where seq1 and seq2 are disjoint and both have GCD=1

**Output:** `10`

---

## Example 2:

**Input:** `nums = [10,20,30]`

**Explanation:**

**GCD Analysis:**
- GCD(10) = 10
- GCD(20) = 20
- GCD(30) = 30
- GCD(10,20) = 10
- GCD(10,30) = 10
- GCD(20,30) = 10
- GCD(10,20,30) = 10

**Valid pairs with equal GCD:**
1. seq1=[10], seq2=[20,30] → GCD(10)=10, GCD(20,30)=10
2. seq1=[20,30], seq2=[10] → GCD(20,30)=10, GCD(10)=10

**Output:** `2`

---

## Example 3:

**Input:** `nums = [1,1,1,1]`

**Explanation:**

**All elements are 1, so GCD is always 1**

**Counting:**
- For 4 elements, each can go to seq1, seq2, or neither
- Total states: 3^4 = 81
- Subtract cases where seq1 or seq2 is empty
- Valid pairs: Many combinations where both subsequences are non-empty

**Calculation:**
- Total ways to partition into 2 non-empty disjoint subsequences
- Formula involves combinatorics: $$\sum_{k=1}^{n-1} \binom{n}{k} = 2^n - 2$$
- But we need ordered pairs, so it's more complex

**Output:** `50`

## Key Insights

- **DP State Space** - Track GCD of both subsequences simultaneously
- **Sparse Storage** - Use Map to avoid storing zero states
- **GCD Properties** - GCD is associative and commutative
- **Modular Arithmetic** - Apply MOD at each step to prevent overflow
- **Three Choices** - Each element can go to seq1, seq2, or neither

---

## Edge Cases

- All elements equal → GCD is that element
- All elements coprime → GCD is always 1
- Array with 1s → Many valid pairs
- Large values → GCD computation efficiency matters
