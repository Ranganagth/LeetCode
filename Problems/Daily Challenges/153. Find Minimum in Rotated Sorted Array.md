[153. Find Minimum in Rotated Sorted Array](https://leetcode.com/problems/find-minimum-in-rotated-sorted-array/)

# Intuition

In a rotated sorted array:

* One half is always sorted
* Minimum lies in the **unsorted half**
* If the current segment is already sorted, leftmost element is minimum

Use binary search.

---

# Approach

Maintain:
```
left = 0
right = n - 1
```
At each step:

Compute:

$$mid = (left + right) // 2
$$
Cases:

1. Current range sorted:

$$nums[left] <= nums[right]
$$
Minimum = `nums[left]`

2. Mid in left sorted portion:
$$
nums[mid] >= nums[left]
$$
Minimum lies right:
$$
left = mid + 1
$$
3. Otherwise:
   Minimum lies left including mid:
$$
right = mid
$$
---

# Complexity

* **Time complexity:**
  $$O(\log n)$$

* **Space complexity:**
  $$O(1)$$

---

# Code

```typescript
function findMin(nums: number[]): number {
    let left = 0;
    let right = nums.length - 1;

    while (left < right) {
        // already sorted
        if (nums[left] < nums[right]) {
            return nums[left];
        }

        const mid = Math.floor((left + right) / 2);

        if (nums[mid] >= nums[left]) {
            left = mid + 1;
        } else {
            right = mid;
        }
    }

    return nums[left];
};

```

# Example Walkthrough

## Example 1: Step by step explanation

Input:

```
[3,4,5,1,2]
```

Iteration 1:

```
left=0 right=4
mid=2 → nums[mid]=5
```

5 ≥ 3 → search right

```
left=3 right=4
```

Now:

```
nums[left]=1 < nums[right]=2
```

Return:

```
1
```
