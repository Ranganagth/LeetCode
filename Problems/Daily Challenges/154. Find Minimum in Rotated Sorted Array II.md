[154. Find Minimum in Rotated Sorted Array II](https://leetcode.com/problems/find-minimum-in-rotated-sorted-array-ii/)

# Intuition

This is similar to rotated sorted array minimum search, but duplicates create ambiguity.

Example:

```
[2,2,2,0,1]
```

When:

```
nums[mid] == nums[right]
```

We cannot determine which side contains the minimum.

Solution:

* Shrink search space:

```
right--
```

---

# Approach

Maintain:

```
left = 0
right = n - 1
```

At each iteration:

### Case 1:

```
nums[mid] < nums[right]
```

Minimum in left half including mid:

```
right = mid
```

### Case 2:

```
nums[mid] > nums[right]
```

Minimum in right half:

```
left = mid + 1
```

### Case 3 (duplicates):

```
nums[mid] == nums[right]
```

Cannot decide:

```
right--
```

---

# Complexity

* **Average time complexity:**
  $$O(\log n)$$

* **Worst-case time complexity (many duplicates):**
  $$O(n)$$

Reason:
For arrays like:

```
[1,1,1,1,1]
```

Only:

```
right--
```

happens repeatedly.

* **Space complexity:**
  $$O(1)$$

---

# Code

```typescript
function findMin(nums: number[]): number {
    let left = 0;
    let right = nums.length - 1;

    while (left < right) {
        const mid = Math.floor((left + right) / 2);

        if (nums[mid] < nums[right]) {
            right = mid;
        } 
        else if (nums[mid] > nums[right]) {
            left = mid + 1;
        } 
        else {
            right--;
        }
    }

    return nums[left];
};

```

## Follow-up: Effect of duplicates

Without duplicates:

$$O(log n)
$$
With duplicates:

$$Worst case O(n)
$$

because equal elements prevent binary search from eliminating half the array.
