[1878. Get Biggest Three Rhombus Sums in a Grid](https://leetcode.com/problems/get-biggest-three-rhombus-sums-in-a-grid/)

# Intuition

A **rhombus** in the grid is a square rotated 45°.
It is defined by:

```
top
left  right
bottom
```

Only the **border cells** are included in the sum.

For each cell `(r,c)` treat it as the **top vertex** of a rhombus.

If the rhombus has radius `k`:

```
top       (r,c)

left      (r+k,c-k)
right     (r+k,c+k)

bottom    (r+2k,c)
```

Boundary constraint:

```
r + 2k < m
c - k >= 0
c + k < n
```

Special case:

```
k = 0 → single cell rhombus
```

Goal:

```
collect all rhombus border sums
return top 3 distinct values
```

Grid size:

```
m,n ≤ 50
```

Brute force over all rhombuses is acceptable.

---

# Approach

### Step 1 - Store unique sums

Use a `Set`.

Add:

```
grid[i][j]
```

for radius `0`.

### Step 2 - Enumerate rhombus sizes

For every `(r,c)` and size `k`:

Validate bounds.

Compute border sum.

Edges:

```
1. top → right
2. right → bottom
3. bottom → left
4. left → top
```

Each edge length:

```
k
```

Traverse diagonally.

### Step 3 - Collect results

Convert set → array.

Sort descending.

Return:

```
first 3 values
```

## Algorithm

For each cell:

```
add single cell sum
```

For each size `k`:

```
if rhombus fits
    compute border
    add to set
```

---

# Complexity

Let:
$m = rows$
$n = columns$

Maximum rhombus size: $≤ min(m,n)/2$
**Enumeration:** $O(m * n * min(m,n))$

**Worst case:** $50 * 50 * 25 ≈ 62k rhombuses$
Each border traversal: **O(k)**

- **Total worst case:** $≈ 1–2 million operations$
	Feasible.

- **Space Complexity:** $O(U)$
	`U` = number of unique rhombus sums.
		At most:
			$m * n * possibleSizes$
	But practically small due to `Set`.

# Code

```typescript
function getBiggestThree(grid: number[][]): number[] {

    const m = grid.length;
    const n = grid[0].length;

    const sums = new Set<number>();

    for (let r = 0; r < m; r++) {
        for (let c = 0; c < n; c++) {

            sums.add(grid[r][c]);

            let k = 1;

            while (
                r + 2 * k < m &&
                c - k >= 0 &&
                c + k < n
            ) {

                let sum = 0;

                let x = r, y = c;

                for (let i = 0; i < k; i++) sum += grid[x + i][y + i];
                for (let i = 0; i < k; i++) sum += grid[x + k + i][y + k - i];
                for (let i = 0; i < k; i++) sum += grid[x + 2 * k - i][y - i];
                for (let i = 0; i < k; i++) sum += grid[x + k - i][y - k + i];

                sums.add(sum);

                k++;
            }
        }
    }

    return Array.from(sums)
        .sort((a, b) => b - a)
        .slice(0, 3);
};

```

# Example Walkthrough

### Example

```
grid =
[
[1,2,3],
[4,5,6],
[7,8,9]
]
```

### Radius 0 Rhombuses

Single cells:

```
1 2 3
4 5 6
7 8 9
```

Distinct sums:

```
1,2,3,4,5,6,7,8,9
```

### Radius 1 Rhombus

Top = `(0,1)`

Shape:

```
   2
 4   6
   8
```

Border sum:

```
2 + 4 + 6 + 8 = 20
```

Add:

```
20
```

### All sums

```
{1,2,3,4,5,6,7,8,9,20}
```

Top 3 descending:

```
20
9
8
```

Result:

```
[20,9,8]
```
