[2126. Destroying Asteroids](https://leetcode.com/problems/destroying-asteroids/)

# Intuition

To maximize the chances of destroying all asteroids, the optimal strategy is to process the **smallest asteroids first**. This way, the planet's mass increases gradually, making it more likely to be able to destroy larger asteroids later. If at any point, the planet's mass is less than an asteroid's mass, it cannot destroy it, and the answer is false.

---

# Approach

1. Sort the `asteroids` array in ascending order.
2. Iterate through the sorted array:
   - If the planet's current mass is **greater than or equal** to the asteroid's mass, destroy it:
     - Increase the planet's mass by the asteroid's mass.
   - Else, return `false` immediately.
3. If all asteroids are processed successfully, return `true`.

---

# Complexity

- **Time complexity:**   $$O(n \log n)$$ for sorting, where \(n\) is the number of asteroids.

- **Space complexity:** 
$$O(1)$$ or $$O(n)$$ depending on the sorting implementation.

---

# Code

```typescript
function asteroidsDestroyed(mass: number, asteroids: number[]): boolean {
    asteroids.sort((a, b) => a - b);
    for (const asteroid of asteroids) {
        if (mass >= asteroid) {
            mass += asteroid;
        } else {
            return false;
        }
    }
    return true;
};

```

# Example walkthrough

## Example 1:
`mass = 10`, `asteroids = [3, 9, 19, 5, 21]`

- Sorted: `[3, 5, 9, 19, 21]`
- Process:
  - mass=10 >= 3 → destroy, mass=13
  - mass=13 >= 5 → destroy, mass=18
  - mass=18 >= 9 → destroy, mass=27
  - mass=27 >= 19 → destroy, mass=46
  - mass=46 >= 21 → destroy, mass=67
- All destroyed → return `true`

## Example 2:
`mass = 5`, `asteroids = [4, 9, 23, 4]`

- Sorted: `[4, 4, 9, 23]`
- Process:
  - mass=5 >= 4 → destroy, mass=9
  - mass=9 >= 4 → destroy, mass=13
  - mass=13 >= 9 → destroy, mass=22
  - mass=22 >= 23? No → cannot destroy last asteroid → return `false`
