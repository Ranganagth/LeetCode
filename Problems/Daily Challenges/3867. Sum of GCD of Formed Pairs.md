[3867. Sum of GCD of Formed Pairs](https://leetcode.com/problems/sum-of-gcd-of-formed-pairs/submissions/2069912423/?envType=daily-question&envId=2026-07-16)

# Intuition

The problem requires us to:
1. Build a `prefixGcd` array where each element is the GCD of `nums[i]` and the maximum value up to index `i`
2. Sort this array
3. Pair smallest with largest elements
4. Sum the GCD of each pair

The key insight is that we need to efficiently compute GCD and handle the pairing logic correctly.

---

# Approach

## Step-by-Step Strategy

1. **Build prefixGcd array:**
   - Track running maximum `mx` as we iterate
   - For each index `i`, compute `gcd(nums[i], mx)`

2. **Sort prefixGcd:**
   - Sort in non-decreasing order

3. **Form pairs and compute sum:**
   - Use two pointers: `left` at start, `right` at end
   - Pair `prefixGcd[left]` with `prefixGcd[right]`
   - Compute GCD of each pair and add to sum
   - Move pointers inward: `left++`, `right--`
   - Stop when `left >= right` (handles odd length automatically)

4. **Helper function:**
   - Implement efficient GCD using Euclidean algorithm

---

# Complexity

- **Time complexity:** $$O(n \log n + n \log M)$$
  - Building prefixGcd: $$O(n \log M)$$ where M is max value (for GCD computation)
  - Sorting: $$O(n \log n)$$
  - Pairing and summing: $$O(n \log M)$$

- **Space complexity:** $$O(n)$$ - for the prefixGcd array

# Code

```typescript
function gcdSum(nums: number[]): number {
    const n = nums.length;
    
    // Helper function to compute GCD using Euclidean algorithm
    const gcd = (a: number, b: number): number => {
        if (b === 0) return a;
        return gcd(b, a % b);
    };
    
    // Step 1: Build prefixGcd array
    const prefixGcd: number[] = [];
    let mx = 0; // Running maximum
    
    for (let i = 0; i < n; i++) {
        mx = Math.max(mx, nums[i]);
        prefixGcd.push(gcd(nums[i], mx));
    }
    
    // Step 2: Sort prefixGcd in non-decreasing order
    prefixGcd.sort((a, b) => a - b);
    
    // Step 3: Form pairs and compute sum
    let sum = 0;
    let left = 0;
    let right = n - 1;
    
    // Pair smallest with largest until pointers meet
    while (left < right) {
        sum += gcd(prefixGcd[left], prefixGcd[right]);
        left++;
        right--;
    }
    
    // If n is odd, the middle element is ignored (handled by left < right condition)
    
    return sum;
};

```

## GCD Computation Examples

**gcd(4, 6):**
```
6 = 4 × 1 + 2
4 = 2 × 2 + 0
→ gcd = 2
```

**gcd(2, 8):**
```
8 = 2 × 4 + 0
→ gcd = 2
```

**gcd(3, 6):**
```
6 = 3 × 2 + 0
→ gcd = 3
```

## Key Insights

- **Running Maximum** - Track max value efficiently in single pass
- **Two-Pointer Pairing** - Pair smallest with largest for optimal pairing
- **Automatic Odd Handling** - `left < right` condition naturally ignores middle element
- **GCD Properties** - gcd(a, b) = gcd(b, a % b) for efficient computation

## Alternative Optimization

For very large arrays, we could optimize GCD computation using:
- **Iterative GCD** instead of recursive (avoid stack overflow)
- **Binary GCD algorithm** for better performance with large numbers

```typescript
// Iterative GCD (alternative)
const gcd = (a: number, b: number): number => {
    while (b !== 0) {
        [a, b] = [b, a % b];
    }
    return a;
};

```
