[3546. Equal Sum Grid Partition I](https://leetcode.com/problems/equal-sum-grid-partition-i/)

# Intuition

A valid cut splits the grid into two parts with equal sum.
So the total sum must be even.

Instead of recomputing sums for every possible cut, accumulate progressively:

* For horizontal cuts → keep adding row sums
* For vertical cuts → keep adding column sums

Check if at any cut point, accumulated sum equals half of total.

---

# Approach

1. Compute total sum of grid.
2. If total is odd → return false.
3. Set target = total / 2.

**Horizontal cut check:**

* Iterate row by row (excluding last row)
* Maintain running sum of rows
* If sum == target → return true

**Vertical cut check:**

* Iterate column by column (excluding last column)
* Maintain running sum of columns
* If sum == target → return true

4. If no valid cut found → return false

---

# Complexity

* **Time complexity:**
  $$O(m \cdot n)$$

* **Space complexity:**
  $$O(1)$$

---

# Code

```typescript [] id="hzp341"
function canPartitionGrid(grid: number[][]): boolean {
    const m = grid.length, n = grid[0].length;
    let total = 0;

    // Total sum
    for (let i = 0; i < m; i++) {
        for (let j = 0; j < n; j++) {
            total += grid[i][j];
        }
    }

    if (total % 2 !== 0) return false;

    const target = total / 2;

    // Horizontal cut
    let sum = 0;
    for (let i = 0; i < m - 1; i++) {
        for (let j = 0; j < n; j++) {
            sum += grid[i][j];
        }
        if (sum === target) return true;
    }

    // Vertical cut
    sum = 0;
    for (let j = 0; j < n - 1; j++) {
        for (let i = 0; i < m; i++) {
            sum += grid[i][j];
        }
        if (sum === target) return true;
    }

    return false;
};

```
