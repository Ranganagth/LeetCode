[3740. Minimum Distance Between Three Equal Elements I](https://leetcode.com/problems/minimum-distance-between-three-equal-elements-i/)

# Intuition

For any triple `(i, j, k)` with `i < j < k`:

$$
|i - j| + |j - k| + |k - i|
= (j - i) + (k - j) + (k - i)
= 2 * (k - i)
$$
So the distance depends **only on the first and last index**, not the middle.

Goal reduces to:

* For each value, find **3 indices**
* Minimize `k - i` (range)
* Answer = `2 * (k - i)`

---

# Approach

1. Group indices by value
2. For each value:

   * If it appears at least 3 times:

     * Slide window of size 3 over indices
     * For indices `[i, j, k]`:
$$distance = 2 * (k - i)$$

     * Track minimum
3. If no valid triple → return -1

---

# Complexity

* **Time complexity:**
  $$O(n)$$

* **Space complexity:**
  $$O(n)$$

---

# Code

```typescript
function minimumDistance(nums: number[]): number {
    const map = new Map<number, number[]>();

    // group indices
    for (let i = 0; i < nums.length; i++) {
        if (!map.has(nums[i])) map.set(nums[i], []);
        map.get(nums[i])!.push(i);
    }

    let ans = Infinity;

    for (const indices of map.values()) {
        if (indices.length < 3) continue;

        for (let i = 0; i + 2 < indices.length; i++) {
            const left = indices[i];
            const right = indices[i + 2];
            ans = Math.min(ans, 2 * (right - left));
        }
    }

    return ans === Infinity ? -1 : ans;
};

```
