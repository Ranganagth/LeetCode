[2946. Matrix Similarity After Cyclic Shifts](https://leetcode.com/problems/matrix-similarity-after-cyclic-shifts/)

# Intuition

Each row undergoes cyclic shifts repeatedly. After `k` steps:

* Even-indexed rows → shifted left by `k`
* Odd-indexed rows → shifted right by `k`

A row returns to original only if shifting by `k` results in the same configuration.
This is equivalent to checking if the row is invariant under that rotation.

---

# Approach

1. Let `n` be number of columns.
2. Reduce shifts using `k % n` (full cycles cancel out).
3. For each row `i`:

   * If `i` is even → left shift by `k`

     * Compare `mat[i][j]` with `mat[i][(j + k) % n]`
   * If `i` is odd → right shift by `k`

     * Compare `mat[i][j]` with `mat[i][(j - k + n) % n]`
4. If all elements match → return true
5. Else → return false

---

# Complexity

* **Time complexity:**
  $$O(m \cdot n)$$

* **Space complexity:**
  $$O(1)$$

---

# Code

```typescript [] id="qwe219"
function areSimilar(mat: number[][], k: number): boolean {
    const m = mat.length;
    const n = mat[0].length;

    k %= n;

    for (let i = 0; i < m; i++) {
        for (let j = 0; j < n; j++) {
            if (i % 2 === 0) {
                // even row → left shift
                if (mat[i][j] !== mat[i][(j + k) % n]) {
                    return false;
                }
            } else {
                // odd row → right shift
                if (mat[i][j] !== mat[i][(j - k + n) % n]) {
                    return false;
                }
            }
        }
    }

    return true;
};

```
