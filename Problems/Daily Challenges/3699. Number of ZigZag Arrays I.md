[3699. Number of ZigZag Arrays I](https://leetcode.com/problems/number-of-zigzag-arrays-i/)

# Intuition

This is a dynamic programming problem where we track the direction of transitions in a ZigZag array:
- `dp0[j]` = number of valid sequences ending at value `j` where the last transition was **increasing** (previous element < current element)
- `dp1[j]` = number of valid sequences ending at value `j` where the last transition was **decreasing** (previous element > current element)

The key constraint is that directions must alternate: if the last transition was increasing, the next must be decreasing, and vice versa.

To optimize, we use prefix sums:
- `sum0[j]` = cumulative sum of `dp0[0...j]`
- `sum1[j]` = cumulative sum of `dp1[0...j]`

This allows us to compute the sum of ranges in O(1) time instead of iterating through all values.

---

# Approach

1. **Initialize:** For the first element, all values in [l, r] can be placed with either direction (both dp0 and dp1 = 1).
2. **For each subsequent position i from 1 to n-1:**
   - For each value `j` in [l, r]:
     - `dp0[j]` (ending with increasing): Sum of all `dp1[k]` where `k < j` (came from smaller values with decreasing direction)
     - `dp1[j]` (ending with decreasing): Sum of all `dp0[k]` where `k > j` (came from larger values with increasing direction)
   - Update prefix sums for the next iteration.
3. **Return:** Sum of all `dp0[j]` and `dp1[j]` for all j in [l, r].

---

# Complexity

- **Time complexity:** $$O(n \times (r-l))$$, as we process n positions and for each position we iterate through the range once.
  
- **Space complexity:** $$O(r)$$ for the DP and prefix sum arrays.

# Code

```typescript
const MOD = 10 ** 9 + 7;

function zigZagArrays(n: number, l: number, r: number): number {
    const dp0 = new Array<number>(r + 1).fill(0);
    const dp1 = new Array<number>(r + 1).fill(0);
    const sum0 = new Array<number>(r + 2).fill(0);
    const sum1 = new Array<number>(r + 2).fill(0);

    // Initialize: first element can be any value with either direction
    for (let i = l; i <= r; i++) {
        dp0[i] = dp1[i] = 1;
        sum0[i] = sum1[i] = i - l + 1;
    }

    // DP for positions 1 to n-1
    for (let i = 1; i < n; i++) {
        for (let j = l; j <= r; j++) {
            // dp0[j]: ending with increasing transition
            // Sum of dp1[k] where k < j (came from smaller values with decreasing)
            dp0[j] = sum1[j - 1];

            // dp1[j]: ending with decreasing transition
            // Sum of dp0[k] where k > j (came from larger values with increasing)
            dp1[j] = (sum0[r] - sum0[j] + MOD) % MOD;
        }

        // Update prefix sums for next iteration
        sum0[l] = dp0[l];
        sum1[l] = dp1[l];
        for (let j = l + 1; j <= r; j++) {
            sum0[j] = (sum0[j - 1] + dp0[j]) % MOD;
            sum1[j] = (sum1[j - 1] + dp1[j]) % MOD;
        }
    }

    // Return total count of valid sequences
    return (sum0[r] + sum1[r]) % MOD;
};

```

# Example walkthrough

## Example 1:
Input: `n = 3, l = 4, r = 5`

**Position 0 (initialization):**
- dp0[4]=1, dp1[4]=1
- dp0[5]=1, dp1[5]=1
- sum0: [0,0,0,0,1,2], sum1: [0,0,0,0,1,2]

**Position 1:**
- j=4: dp0[4]=sum1[3]=0, dp1[4]=sum0[5]-sum0[4]=2-1=1
- j=5: dp0[5]=sum1[4]=1, dp1[5]=sum0[5]-sum0[5]=0
- New sums: sum0[4]=0, sum0[5]=1, sum1[4]=1, sum1[5]=1

**Position 2:**
- j=4: dp0[4]=sum1[3]=0, dp1[4]=sum0[5]-sum0[4]=1-0=1
- j=5: dp0[5]=sum1[4]=1, dp1[5]=sum0[5]-sum0[5]=0
- Final sum: sum0[5]+sum1[5]=1+1=2

**Output:** `2`
