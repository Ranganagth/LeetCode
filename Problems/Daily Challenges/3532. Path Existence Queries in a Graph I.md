[3532. Path Existence Queries in a Graph I](https://leetcode.com/problems/path-existence-queries-in-a-graph-i/)

# Intuition

We need to determine if there's a path between two nodes in a graph where edges exist based on the difference between node values.

**Key insights:**
1. An edge exists between nodes i and j if `|nums[i] - nums[j]| <= maxDiff`.
2. Since `nums` is sorted, we can efficiently find which nodes are connected.
3. Use **Union-Find (Disjoint Set Union)** to group connected nodes.
4. For each query, check if both nodes are in the same connected component.

**Why Union-Find:**
- Building the graph explicitly would be O(n²) in the worst case.
- Union-Find allows us to efficiently group connected nodes in O(n log n).
- Answering queries becomes O(α(n)) per query (nearly O(1)).

**Algorithm:**
1. Create a sorted list of (value, original_index) pairs.
2. Use Union-Find to connect consecutive nodes in the sorted list if their difference is ≤ maxDiff.
3. For each query, check if both nodes are in the same component using `find()`.

---

# Approach

1. **Create sorted pairs**: Store (nums[i], i) and sort by value.
2. **Union-Find initialization**: Create a parent array for each node.
3. **Build connected components**:
   - Iterate through sorted pairs.
   - If the difference between consecutive values is ≤ maxDiff, union them.
4. **Answer queries**:
   - For each query [u, v], check if `find(u) === find(v)`.

---

# Complexity

- **Time complexity:** 
  $$O(n \log n + q \cdot \alpha(n))$$, where `n` is the number of nodes, `q` is the number of queries, and `α(n)` is the inverse Ackermann function (nearly constant).

- **Space complexity:** 
  $$O(n)$$ for the Union-Find structure and sorted pairs.

# Implementation

```typescript
function pathExistenceQueries(
    n: number,
    nums: number[],
    maxDiff: number,
    queries: number[][]
): boolean[] {
    // Union-Find data structure
    const parent = Array.from({ length: n }, (_, i) => i);
    const rank = new Array(n).fill(0);

    const find = (x: number): number => {
        if (parent[x] !== x) {
            parent[x] = find(parent[x]); // Path compression
        }
        return parent[x];
    };

    const union = (x: number, y: number): void => {
        const rootX = find(x);
        const rootY = find(y);

        if (rootX === rootY) return;

        // Union by rank
        if (rank[rootX] < rank[rootY]) {
            parent[rootX] = rootY;
        } else if (rank[rootX] > rank[rootY]) {
            parent[rootY] = rootX;
        } else {
            parent[rootY] = rootX;
            rank[rootX]++;
        }
    };

    // Create sorted pairs: (value, original_index)
    const sortedPairs: [number, number][] = nums.map((val, idx) => [val, idx]);
    sortedPairs.sort((a, b) => a[0] - b[0]);

    // Build connected components
    for (let i = 0; i < n - 1; i++) {
        const [val1, idx1] = sortedPairs[i];
        const [val2, idx2] = sortedPairs[i + 1];

        // If consecutive values in sorted order differ by at most maxDiff, connect them
        if (val2 - val1 <= maxDiff) {
            union(idx1, idx2);
        }
    }

    // Answer queries
    const answer: boolean[] = [];
    for (const [u, v] of queries) {
        answer.push(find(u) === find(v));
    }

    return answer;
};

```

# Example walkthrough
## Example 1:
**Input:** `n = 2, nums = [1,3], maxDiff = 1, queries = [[0,0],[0,1]]`

```
parent = [0, 1], rank = [0, 0]

sortedPairs = [[1, 0], [3, 1]]
Already sorted

Build components:
i = 0: val1 = 1, idx1 = 0, val2 = 3, idx2 = 1
  val2 - val1 = 3 - 1 = 2 > maxDiff = 1
  Don't union

parent = [0, 1]

Query [0, 0]: find(0) = 0, find(0) = 0 → true ✓
Query [0, 1]: find(0) = 0, find(1) = 1 → false ✓
```

**Output:** `[true, false]` ✓

## Example 2:
**Input:** `n = 4, nums = [2,5,6,8], maxDiff = 2, queries = [[0,1],[0,2],[1,3],[2,3]]`

```
parent = [0, 1, 2, 3], rank = [0, 0, 0, 0]

sortedPairs = [[2, 0], [5, 1], [6, 2], [8, 3]]
Already sorted

Build components:
i = 0: val1 = 2, idx1 = 0, val2 = 5, idx2 = 1
  5 - 2 = 3 > 2, don't union

i = 1: val1 = 5, idx1 = 1, val2 = 6, idx2 = 2
  6 - 5 = 1 ≤ 2, union(1, 2)
  parent = [0, 1, 1, 3]

i = 2: val1 = 6, idx1 = 2, val2 = 8, idx2 = 3
  8 - 6 = 2 ≤ 2, union(2, 3)
  find(2) = 1, find(3) = 3
  union(1, 3)
  parent = [0, 1, 1, 1]

Query [0, 1]: find(0) = 0, find(1) = 1 → false ✓
Query [0, 2]: find(0) = 0, find(2) = 1 → false ✓
Query [1, 3]: find(1) = 1, find(3) = 1 → true ✓
Query [2, 3]: find(2) = 1, find(3) = 1 → true ✓
```

**Output:** `[false, false, true, true]` ✓

# Key Points

- 🔑 **Sorted array**: Since nums is sorted, consecutive elements in sorted order are the only ones we need to check.
- 🔑 **Union-Find**: Efficiently groups connected nodes and answers connectivity queries.
- 🔑 **Path compression**: Optimizes `find()` operations.
- 🔑 **Union by rank**: Keeps the tree balanced for better performance.
- 🔑 **Transitive connectivity**: If A connects to B and B connects to C, then A and C are in the same component.

> This solution is optimal and demonstrates advanced graph connectivity techniques!
