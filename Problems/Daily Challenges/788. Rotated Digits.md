[788. Rotated Digits](https://leetcode.com/problems/rotated-digits/)

# Intuition

A number is **valid** if all digits can rotate.
A number is **good** if:

* It is valid
* At least one digit changes after rotation

Valid digits:

```
0,1,8 → same
2,5,6,9 → change
```

Invalid digits:

```
3,4,7
```

So:

* Reject if any digit ∈ {3,4,7}
* Accept if at least one digit ∈ {2,5,6,9}

---

# Approach

1. Iterate from `1 → n`
2. For each number:

   * Check digits:

     * If contains invalid → skip
     * If contains changing digit → mark as good
3. Count all good numbers

---

# Complexity

* **Time complexity:**
  $$O(n \cdot d)$$ (d = digits ≤ 5)

* **Space complexity:**
  $$O(1)$$

---

# Code

```typescript
function rotatedDigits(n: number): number {
    const invalid = new Set([3, 4, 7]);
    const change = new Set([2, 5, 6, 9]);

    let count = 0;

    for (let i = 1; i <= n; i++) {
        let x = i;
        let isValid = true;
        let isGood = false;

        while (x > 0) {
            const d = x % 10;

            if (invalid.has(d)) {
                isValid = false;
                break;
            }

            if (change.has(d)) {
                isGood = true;
            }

            x = Math.floor(x / 10);
        }

        if (isValid && isGood) count++;
    }

    return count;
};

```
