[1871. Jump Game VII](https://leetcode.com/problems/jump-game-vii/)

# Intuition
Since we need to determine if it's possible to reach the last index from the start, and movement is constrained by `minJump` and `maxJump`, a straightforward approach involves exploring reachable indices efficiently. Naively checking all possible jumps at each step would be costly, especially for large strings. Instead, we can leverage a **sliding window** or **prefix sum** technique to efficiently determine reachability without redundant computations.

---

# Approach
- Use a **boolean array** `reachable` to mark whether each index can be reached.
- Initialize `reachable[0] = true` since we start at index 0.
- Maintain a **window sum** that tracks how many of the previous `maxJump - minJump + 1` positions are reachable.
- Iterate through the string:
  - For each index `i`, if `reachable[i]` is true, then all indices from `i + minJump` to `i + maxJump` (within bounds) can potentially be reached.
  - Use the window sum to quickly determine if `i` is reachable based on previous reachable positions.
- If the last index is reachable, return `true`; otherwise, `false`.

This approach ensures an **O(n)** time complexity by avoiding repeated checks for each position.

---

# Complexity
- **Time complexity:** 
  $$O(n)$$ - Each index is processed once, with constant-time updates using sliding window sum.
- **Space complexity:** 
  $$O(n)$$ - For the `reachable` array.

---

# Code

```typescript
function canReach(s: string, minJump: number, maxJump: number): boolean {
    const n = s.length;
    const reachable = new Array<boolean>(n).fill(false);
    reachable[0] = true;
    let windowSum = 0;

    for (let i = 1; i < n; i++) {
        // Remove the position that is out of the current window
        if (i - maxJump - 1 >= 0 && reachable[i - maxJump - 1]) {
            windowSum -= 1;
        }
        // Add the position that enters the window
        if (i - minJump >= 0 && reachable[i - minJump]) {
            windowSum += 1;
        }
        // If there's at least one reachable position in the window, mark current as reachable
        if (windowSum > 0 && s[i] === '0') {
            reachable[i] = true;
        }
    }
    return reachable[n - 1];
};

```

# Example walkthrough

## Example 1:
`s = "011010"`, `minJump = 2`, `maxJump = 3`

| Index | s[i] | reachable[i] | Explanation |
|--------|--------|--------------|--------------|
| 0 | '0' | true | Starting point |
| 1 | '1' | false | Cannot jump here (blocked) |
| 2 | '1' | false | Blocked, no reachable path |
| 3 | '0' | true | From index 0, jump 3 (within min and max), reachable |
| 4 | '1' | false | Blocked |
| 5 | '0' | true | From index 3, jump 2 to reach index 5 |

**Result:** `true`
