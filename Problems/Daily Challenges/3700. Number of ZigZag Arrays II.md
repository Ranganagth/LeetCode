[3700. Number of ZigZag Arrays II](https://leetcode.com/problems/number-of-zigzag-arrays-ii/)

# Intuition
This problem requires counting valid ZigZag arrays of potentially huge length (up to 10^9). Since we can't iterate through all positions, we use **matrix exponentiation** to compute the result in O(log n) time.

The key insight is:
- Track states based on the current value and the direction of the last transition.
- Build a transition matrix that encodes valid transitions between states.
- Use matrix exponentiation to compute the state after n-1 transitions.
- Sum all final states to get the answer.

---

# Approach
1. **Define states:**
   - States 0 to m-1: ending at value `l+i` with the last transition being **increasing**.
   - States m to 2m-1: ending at value `l+i` with the last transition being **decreasing**.

2. **Build transition matrix:**
   - From state `i` (increasing to value i), we can only go to states `j+m` where `j < i` (decreasing to a smaller value).
   - From state `i+m` (decreasing to value i), we can only go to states `j` where `j > i` (increasing to a larger value).

3. **Initialize DP:**
   - Start with all states having count 1 (all values can be placed at position 1 with either direction).

4. **Apply matrix exponentiation:**
   - Multiply the DP vector by the transition matrix n-1 times using fast exponentiation.

5. **Sum all final states** to get the total count.

---

# Complexity
- **Time complexity:** 
  $$O(m^3 \log n)$$, where `m = r - l + 1` (at most 75) and we perform O(log n) matrix multiplications.

- **Space complexity:** 
  $$O(m^2)$$ for the transition matrix.

# Code Explanation

```typescript
const MOD = 1_000_000_007n;

class Matrix {
    data: BigInt64Array;

    constructor(
        public n: number,
        public m: number,
    ) {
        this.data = new BigInt64Array(n * m);
    }

    // Get element at position (i, j)
    get(i: number, j: number): bigint {
        return this.data[i * this.m + j];
    }

    // Set element at position (i, j)
    set(i: number, j: number, val: bigint) {
        this.data[i * this.m + j] = val;
    }

    // Matrix multiplication
    mul(b: Matrix): Matrix {
        const res = new Matrix(this.n, b.m);

        for (let i = 0; i < this.n; i++) {
            for (let k = 0; k < this.m; k++) {
                const r = this.get(i, k);
                if (r === 0n) continue; // Skip zero elements for efficiency

                for (let j = 0; j < b.m; j++) {
                    res.set(i, j, (res.get(i, j) + r * b.get(k, j)) % MOD);
                }
            }
        }
        return res;
    }

    // Fast matrix exponentiation
    powMul(exp: bigint, res: Matrix): Matrix {
        let base = new Matrix(this.n, this.m);
        base.data = new BigInt64Array(this.data);

        while (exp > 0n) {
            if ((exp & 1n) === 1n) {
                res = res.mul(base);
            }
            base = base.mul(base);
            exp >>= 1n;
        }

        return res;
    }
}

function zigZagArrays(n: number, l: number, r: number): number {
    const m = r - l + 1;
    if (n === 1) return m;

    // Build transition matrix
    let u = new Matrix(2 * m, 2 * m);

    for (let i = 0; i < m; i++) {
        // From increasing state i, go to decreasing states j where j < i
        for (let j = 0; j < i; j++) {
            u.set(i, j + m, 1n);
        }
        // From decreasing state i, go to increasing states j where j > i
        for (let j = i + 1; j < m; j++) {
            u.set(i + m, j, 1n);
        }
    }

    // Initialize DP: all states start with count 1
    let dp = new Matrix(1, 2 * m);
    for (let i = 0; i < 2 * m; i++) {
        dp.set(0, i, 1n);
    }

    // Apply transition matrix n-1 times
    dp = u.powMul(BigInt(n - 1), dp);

    // Sum all final states
    let ans = 0n;
    for (let i = 0; i < 2 * m; i++) {
        ans = (ans + dp.get(0, i)) % MOD;
    }

    return Number(ans);
};

```

# Example walkthrough
## Example 1:
**Input:** `n = 3, l = 4, r = 5`

- m = 2
- Transition matrix (4x4):
  ```
  [0, 0, 0, 0] (from state 0, can't go anywhere)
  [0, 0, 1, 0] (from state 1, go to state 2+1=3)
  [1, 0, 0, 0] (from state 2, go to state 0)
  [1, 1, 0, 0] (from state 3, go to states 0,1)
  ```
- Initial DP: [1, 1, 1, 1]
- After 2 transitions: [1, 1, 0, 0] (simplified)
- Sum: 2

**Output:** `2` 

## Example 2:
**Input:** `n = 3, l = 1, r = 3`

- m = 3
- Build 6x6 transition matrix
- Apply for n-1 = 2 steps
- Sum all final states

**Output:** `10`

This optimized solution efficiently handles large values of n using matrix exponentiation with BigInt for precision.
