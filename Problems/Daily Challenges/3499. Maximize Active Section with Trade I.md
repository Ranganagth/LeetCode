# Intuition

The key insight is understanding what the trade operation actually accomplishes:

1. When we remove a block of '1's surrounded by '0's, we're essentially merging two adjacent blocks of '0's
2. When we then convert a block of '0's surrounded by '1's to '1's, we're filling in a gap between '1's

The optimal strategy is to find two consecutive blocks of '0's (which have a block of '1's between them) and convert both to '1's. This effectively removes the '1' block between them and fills both '0' blocks.

The net gain is: (size of first 0-block) + (size of second 0-block)

---

# Approach

1. **Count original 1's:** This is our baseline

2. **Find all blocks of 0's:** Parse the string to identify contiguous blocks of '0's and their lengths

3. **Find best pair:** Look for two consecutive '0' blocks (which means there's a '1' block between them). The gain from trading is the sum of their lengths.

4. **Edge cases:**
   - If there are fewer than 2 blocks of '0's, no beneficial trade exists
   - The augmented '1's at the ends ensure edge blocks can participate

5. **Return result:** Original count + best gain

---

# Complexity

- **Time complexity:** O(n) - single pass to count 1's and identify 0-blocks

- **Space complexity:** O(k) where k is the number of 0-blocks (at most n/2)

---

# Code

```typescript
function maxActiveSectionsAfterTrade(s: string): number {
    const n = s.length;
    
    // Count original 1's
    let cnt1 = 0;
    for (const c of s) {
        if (c === "1") cnt1++;
    }

    // Find all blocks of 0's and their lengths
    const zeroBlocks: number[] = [];
    let i = 0;
    
    while (i < n) {
        const start = i;
        // Find end of current block
        while (i < n && s[i] === s[start]) {
            i++;
        }
        // If this block is 0's, record its length
        if (s[start] === "0") {
            zeroBlocks.push(i - start);
        }
    }

    const m = zeroBlocks.length;
    
    // If less than 2 zero blocks, no beneficial trade possible
    if (m < 2) {
        return cnt1;
    }

    // Find best gain by trying consecutive pairs of zero blocks
    let bestGain = 0;
    for (let j = 0; j < m - 1; j++) {
        // Gain from converting both zero blocks to 1's
        // (This implicitly removes the 1-block between them)
        bestGain = Math.max(bestGain, zeroBlocks[j] + zeroBlocks[j + 1]);
    }

    return cnt1 + bestGain;
};

```

## Example 2:

**Input:** `s = "0100"`

**Step 1: Count 1's**
- cnt1 = 1

**Step 2: Find zero blocks**
- Parse: "0" | "1" | "00"
- Block at index 0: "0" (length 1)
- Block at index 2-3: "00" (length 2)
- zeroBlocks = [1, 2]

**Step 3: Find best gain**
- Pair (0,1): zeroBlocks[0] + zeroBlocks[1] = 1 + 2 = 3
- bestGain = 3

**Step 4: Calculate result**
- Result = cnt1 + bestGain = 1 + 3 = 4

**Explanation:**
- Original: "0100" has 1 one
- Trade removes the "1" and converts both "0" blocks to "1"
- Result: "1111" has 4 ones

**Output:** `4`

## Why This Works

**Key observation:** When we have the pattern `0...0 1...1 0...0`:
- The augmented string treats this as: `1 0...0 1...1 0...0 1`
- The middle `1...1` block is surrounded by `0`'s
- The two `0...0` blocks are surrounded by `1`'s
- Trading removes the middle `1...1` and fills both `0...0` blocks
- Net gain = length(first 0-block) + length(second 0-block)

**Why consecutive blocks?**
- Two consecutive 0-blocks in the original string means there's exactly one 1-block between them
- This 1-block can be removed (it's surrounded by 0's in augmented string)
- Both 0-blocks can be filled (they're surrounded by 1's in augmented string)

## Edge Cases

| Case | Input | Zero Blocks | Best Gain | Output |
|------|-------|-------------|-----------|--------|
| All 1's | "1111" | [] | 0 | 4 |
| All 0's | "0000" | [4] | 0 | 0 |
| Single 0 | "10001" | [3] | 0 | 2 |
| Alternating | "010101" | [1,1,1,1] | 2 | 5 |
| No trade | "100" | [2] | 0 | 1 |

---

## Key Insights

- **Pattern recognition:** The problem is about merging sections by removing barriers
- **Consecutive blocks:** Only consecutive 0-blocks can be merged in one trade
- **Augmentation effect:** The '1's at ends ensure edge cases work correctly
- **Greedy choice:** Taking the maximum sum of consecutive pairs is optimal
- **Simple solution:** No need to simulate the actual trade operation

> This solution is elegant because it directly computes the answer without simulating the complex trade mechanics!
