[3719. Longest Balanced Subarray I](https://leetcode.com/problems/longest-balanced-subarray-i/)
# Intuition

A subarray is balanced when

```
#distinct_even == #distinct_odd
```

Since `n ≤ 1500`, checking all subarrays is feasible if each subarray can be evaluated efficiently.

For each starting index `i`, expand the subarray to the right while maintaining:

* a set of distinct even numbers
* a set of distinct odd numbers

At each extension:

```
if evenSet.size == oddSet.size
    update answer
```

This yields an `O(n^2)` algorithm, which is optimal for the given constraints.

---

# Approach

1. For every start index `i`:

   * Initialize empty `evenSet` and `oddSet`.
2. Extend `j` from `i` to `n-1`:

   * Insert `nums[j]` into the appropriate set (even or odd).
   * If sizes match, update the maximum length.
3. Return the maximum length found.

---

# Complexity

* **Time:** `O(n^2)` (≈ 2.25M operations worst case)
* **Space:** `O(n)` for sets

---

# Code

```ts
function longestBalanced(nums: number[]): number {
    const n = nums.length;
    let ans = 0;

    for (let i = 0; i < n; i++) {
        const evenSet = new Set<number>();
        const oddSet = new Set<number>();

        for (let j = i; j < n; j++) {
            if (nums[j] % 2 === 0) {
                evenSet.add(nums[j]);
            } else {
                oddSet.add(nums[j]);
            }

            if (evenSet.size === oddSet.size) {
                ans = Math.max(ans, j - i + 1);
            }
        }
    }

    return ans;
};

```

# Example Walkthrough

### Example

```
nums = [1,2,3,2]
```

Start `i = 0`

| subarray  | distinct even | distinct odd | balanced       |
| --------- | ------------- | ------------ | -------------- |
| [1]       | {}            | {1}          | no             |
| [1,2]     | {2}           | {1}          | yes → length 2 |
| [1,2,3]   | {2}           | {1,3}        | no             |
| [1,2,3,2] | {2}           | {1,3}        | no             |

Start `i = 1`

| subarray | distinct even | distinct odd | balanced              |
| -------- | ------------- | ------------ | --------------------- |
| [2]      | {2}           | {}           | no                    |
| [2,3]    | {2}           | {3}          | yes → length 2        |
| [2,3,2]  | {2}           | {3}          | yes → length 3 (best) |

Answer:

```
3
```

---

## Key Insight

Balance depends on **distinct parity counts**, not element frequency.
Maintaining two distinct-value sets while expanding the window allows direct evaluation of every candidate subarray efficiently within quadratic time.
