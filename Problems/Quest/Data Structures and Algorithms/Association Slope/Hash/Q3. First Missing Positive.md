# Intuition

The key insight is that for an array of length `n`, the answer must be in the range `[1, n+1]`. 

**Why?**
- If all numbers 1 to n are present, the answer is n+1.
- If any number in [1, n] is missing, that's the answer.
- Numbers outside [1, n] are irrelevant.

**Strategy:**
Use the array itself as a hash table by placing each number in its "correct" position:
- Number `k` should be at index `k-1` (if 1 ≤ k ≤ n).
- After rearranging, scan the array to find the first index where the number doesn't match.

This achieves O(n) time and O(1) space (modifying the input array).

---

# Approach

1. **First pass**: Place each number in its correct position.
   - For each number `nums[i]`, if `1 ≤ nums[i] ≤ n` and `nums[i]` is not at index `nums[i]-1`, swap it to the correct position.
   - Skip duplicates and out-of-range numbers.

2. **Second pass**: Find the first missing positive.
   - Scan the array and find the first index `i` where `nums[i] ≠ i+1`.
   - Return `i+1`.
   - If all positions are correct, return `n+1`.

---

# Complexity

- **Time complexity:** 
  $$O(n)$$ - Each number is moved to its correct position at most once, so two passes through the array.

- **Space complexity:** 
  $$O(1)$$ - Only using the input array for storage, no extra data structures.

---

# Implementation

```typescript
function firstMissingPositive(nums: number[]): number {
    const n = nums.length;

    // First pass: place each number in its correct position
    for (let i = 0; i < n; i++) {
        // Keep swapping until nums[i] is in the correct position or can't be placed
        while (
            nums[i] > 0 &&
            nums[i] <= n &&
            nums[nums[i] - 1] !== nums[i]
        ) {
            // Swap nums[i] with nums[nums[i] - 1]
            const correctIndex = nums[i] - 1;
            [nums[i], nums[correctIndex]] = [nums[correctIndex], nums[i]];
        }
    }

    // Second pass: find the first missing positive
    for (let i = 0; i < n; i++) {
        if (nums[i] !== i + 1) {
            return i + 1;
        }
    }

    // If all positions are correct, return n + 1
    return n + 1;
};

```

# Example walkthrough
## Example 1:
**Input:** `nums = [1,2,0]`

```
First pass (place numbers in correct positions):
i = 0: nums[0] = 1
  1 is in range [1,3] and should be at index 0
  Already in correct position

i = 1: nums[1] = 2
  2 is in range [1,3] and should be at index 1
  Already in correct position

i = 2: nums[2] = 0
  0 is not in range [1,3]
  Skip

Array after first pass: [1,2,0]

Second pass (find first missing):
i = 0: nums[0] = 1 === 0+1
i = 1: nums[1] = 2 === 1+1
i = 2: nums[2] = 0 ≠ 2+1
  Return 3
```

**Output:** `3`

  Swap nums[1] and nums[0]: [1, -1, 3, 4]
  nums[1] = -1 (not in range, skip)

i = 2: nums[2] = 3

Array after first pass: [1, -1, 3, 4]

**Output:** `2`

# Key Points
- **Range constraint**: Answer is always in [1, n+1].
- **In-place rearrangement**: Use the array itself as a hash table.
- **Swap logic**: Only swap if the number is in range and not already in the correct position.
- **Duplicate handling**: The condition `nums[nums[i] - 1] !== nums[i]` prevents infinite loops with duplicates.
