# Intuition

We need to find two numbers that add up to the target. The key insight is to use a **hash map** to store numbers we've seen and their indices.

For each number, we check if its complement (`target - num`) exists in the hash map. If it does, we've found our pair. If not, we add the current number to the map.

**Why this works:**
- We only need one pass through the array.
- Hash map lookup is O(1) on average.
- We avoid the O(n²) brute force approach.

---

# Approach

1. Create a hash map to store `{value: index}` pairs.
2. Iterate through the array:
   - Calculate the complement: `complement = target - nums[i]`.
   - Check if the complement exists in the hash map.
   - If yes, return the indices of the complement and current number.
   - If no, add the current number and its index to the hash map.
3. Return an empty array if no solution is found (though the problem guarantees one exists).

---

# Complexity

- **Time complexity:** 
  $$O(n)$$, where `n` is the length of the array. We iterate through the array once.

- **Space complexity:** 
  $$O(n)$$ for the hash map in the worst case.

---

# Implementation

```typescript
function twoSum(nums: number[], target: number): number[] {
    // Hash map to store {value: index}
    const map = new Map<number, number>();

    for (let i = 0; i < nums.length; i++) {
        const complement = target - nums[i];

        // Check if complement exists in the map
        if (map.has(complement)) {
            return [map.get(complement)!, i];
        }

        // Add current number and its index to the map
        map.set(nums[i], i);
    }

    // No solution found (shouldn't reach here based on problem constraints)
    return [];
};

```

---

# Alternative Implementation (Using Object)

```typescript
function twoSum(nums: number[], target: number): number[] {
    const map: { [key: number]: number } = {};

    for (let i = 0; i < nums.length; i++) {
        const complement = target - nums[i];

        if (complement in map) {
            return [map[complement], i];
        }

        map[nums[i]] = i;
    }

    return [];
};

```

# Example walkthrough

## Example 1:
**Input:** `nums = [2,7,11,15], target = 9`

```
i = 0: nums[0] = 2
  complement = 9 - 2 = 7
  map.has(7)? No
  map = {2: 0}

i = 1: nums[1] = 7
  complement = 9 - 7 = 2
  map.has(2)? Yes → return [map.get(2), 1] = [0, 1]
```

**Output:** `[0,1]`

# Comparison with Other Approaches

| Approach | Time | Space | Notes |
|----------|------|-------|-------|
| **Hash Map** | O(n) | O(n) | Optimal, single pass |
| **Brute Force** | O(n²) | O(1) | Too slow |
| **Two Pointers** | O(n log n) | O(1) | Requires sorted array |

---

# Why Hash Map is Better than Two Pointers
- **Hash Map**: O(n) time, works with unsorted arrays, returns original indices.
- **Two Pointers**: O(n log n) time (due to sorting), loses original indices, requires additional work to recover them.

---

# Key Points
- **Complement calculation**: `complement = target - nums[i]`
- **Check before adding**: Always check if complement exists before adding the current number.
- **Single pass**: We only iterate through the array once.
- **Hash map efficiency**: O(1) average lookup time.

> This is the optimal solution for the two-sum problem and is commonly asked in technical interviews.
