# Intuition

To reverse a linked list, we need to change the direction of all pointers. Each node's `next` pointer should point to the previous node instead of the next node.

We can solve this in two ways:
1. **Iterative approach**: Use three pointers to track previous, current, and next nodes.
2. **Recursive approach**: Reverse the rest of the list first, then adjust pointers.

---

# Approach 1: Iterative (Recommended)

1. Initialize three pointers: `prev = null`, `current = head`, `next = null`.
2. Traverse the list:
   - Save the next node before changing the pointer.
   - Reverse the current node's pointer to point to the previous node.
   - Move `prev` and `current` one step forward.
3. Return `prev` as the new head.

---

# Approach 2: Recursive

1. Base case: if the node is null or has no next node, return it.
2. Recursively reverse the rest of the list.
3. Adjust pointers: make the next node point back to the current node.
4. Set the current node's next to null.
5. Return the new head.

---

# Complexity Analysis

## Iterative:
- **Time complexity:** $$O(n)$$, where `n` is the number of nodes.
- **Space complexity:** $$O(1)$$ - constant extra space.

## Recursive:
- **Time complexity:** $$O(n)$$, where `n` is the number of nodes.
- **Space complexity:** $$O(n)$$ - due to the recursion call stack.

---

# Implementation 1: Iterative (Optimal)

```typescript
/**
 * Definition for singly-linked list.
 * class ListNode {
 * val: number
 * next: ListNode | null
 * constructor(val?: number, next?: ListNode | null) {
 * this.val = (val===undefined ? 0 : val)
 * this.next = (next===undefined ? null : next)
 * }
 * }
 */

function reverseList(head: ListNode | null): ListNode | null {
    let prev: ListNode | null = null;
    let current = head;

    while (current !== null) {
        // Save the next node
        const next = current.next;

        // Reverse the pointer
        current.next = prev;

        // Move prev and current one step forward
        prev = current;
        current = next;
    }

    // prev is now the new head
    return prev;
};

```

---

# Implementation 2: Recursive

```typescript
/**
 * Definition for singly-linked list.
 * class ListNode {
 * val: number
 * next: ListNode | null
 * constructor(val?: number, next?: ListNode | null) {
 * this.val = (val===undefined ? 0 : val)
 * this.next = (next===undefined ? null : next)
 * }
 * }
 */

function reverseList(head: ListNode | null): ListNode | null {
    // Base case: empty list or single node
    if (head === null || head.next === null) {
        return head;
    }

    // Recursively reverse the rest of the list
    const newHead = reverseList(head.next);

    // Adjust pointers: make the next node point back to current
    head.next.next = head;

    // Set current node's next to null
    head.next = null;

    return newHead;
};

```

# Example walkthrough
## Example 1:
**Input:** `head = [1,2,3,4,5]`

### Iterative approach:
```
Initial: prev = null, current = 1

Step 1:
  next = 2
  1.next = null
  prev = 1, current = 2
  List: null ← 1 2 → 3 → 4 → 5

Step 2:
  next = 3
  2.next = 1
  prev = 2, current = 3
  List: null ← 1 ← 2 3 → 4 → 5

Step 3:
  next = 4
  3.next = 2
  prev = 3, current = 4
  List: null ← 1 ← 2 ← 3 4 → 5

Step 4:
  next = 5
  4.next = 3
  prev = 4, current = 5
  List: null ← 1 ← 2 ← 3 ← 4 5

Step 5:
  next = null
  5.next = 4
  prev = 5, current = null
  List: null ← 1 ← 2 ← 3 ← 4 ← 5

Return prev = 5
```

**Output:** `[5,4,3,2,1]`

### Recursive approach:
```
reverseList(1):
  reverseList(2):
    reverseList(3):
      reverseList(4):
        reverseList(5):
          reverseList(null):
            return null
          newHead = null
          5.next.next = 5 (but 5.next is null, so this doesn't apply)
          5.next = null
          return 5
        newHead = 5
        4.next.next = 4 → 5.next = 4
        4.next = null
        return 5
      newHead = 5
      3.next.next = 3 → 4.next = 3
      3.next = null
      return 5
    newHead = 5
    2.next.next = 2 → 3.next = 2
    2.next = null
    return 5
  newHead = 5
  1.next.next = 1 → 2.next = 1
  1.next = null
  return 5
```

**Output:** `[5,4,3,2,1]`

## Example 2:
**Input:** `head = [1,2]`

### Iterative:
```
Step 1: prev = null, current = 1, next = 2
  1.next = null, prev = 1, current = 2

Step 2: next = null
  2.next = 1, prev = 2, current = null

Return prev = 2
```

**Output:** `[2,1]`

## Example 3:
**Input:** `head = []`

- head === null → return null

**Output:** `[]`

# Comparison

| Aspect | Iterative | Recursive |
|--------|-----------|-----------|
| **Time** | O(n) | O(n) |
| **Space** | O(1) | O(n) |
| **Readability** | Clear | Elegant |
| **Stack Overflow Risk** | No | Yes (for very large lists) |

---

# Recommendation

Use the **iterative approach** for production code as it's more efficient with O(1) space complexity and no risk of stack overflow. The recursive approach is good for understanding the concept and for interviews when asked to implement both.
