# Intuition

We need to find the next greater element for each node in a linked list. This is similar to the "Next Greater Element" problem, but with a linked list.

**Key insight:**
- We can't look ahead in a linked list efficiently.
- **Solution**: Traverse the list to collect all values, then use a **monotonic stack** to find the next greater element for each position.
- Alternatively, use a **reverse traversal with a stack** to process nodes from right to left.

**Approach 1 (Recommended):**
1. Convert linked list to an array.
2. Use a monotonic decreasing stack to find the next greater element for each position.

**Approach 2:**
1. Reverse traverse using recursion or by converting to array first.
2. Use a stack to track potential next greater elements.

---

# Approach 1: Array + Monotonic Stack

1. Convert the linked list to an array of values.
2. Use a **monotonic decreasing stack**:
   - For each element, pop all smaller elements from the stack.
   - The top of the stack (if exists) is the next greater element.
   - Push the current element onto the stack.
3. Return the result array.

---

# Complexity

- **Time complexity:** 
  $$O(n)$$, where `n` is the number of nodes. Each element is pushed and popped from the stack once.

- **Space complexity:** 
  $$O(n)$$ for the array and stack.

# Implementation 1: Array + Monotonic Stack (Recommended)

```typescript
/**
 * Definition for singly-linked list.
 * class ListNode {
 * val: number
 * next: ListNode | null
 * constructor(val?: number, next?: ListNode | null) {
 * this.val = (val===undefined ? 0 : val)
 * this.next = (next===undefined ? null : next)
 * }
 * }
 */

function nextLargerNodes(head: ListNode | null): number[] {
    // Convert linked list to array
    const values: number[] = [];
    let current = head;
    while (current !== null) {
        values.push(current.val);
        current = current.next;
    }

    const n = values.length;
    const result = new Array(n).fill(0);

    // Monotonic decreasing stack: stores indices
    const stack: number[] = [];

    for (let i = 0; i < n; i++) {
        // Pop all elements smaller than current value
        while (stack.length > 0 && values[stack[stack.length - 1]] < values[i]) {
            const index = stack.pop()!;
            result[index] = values[i];
        }

        // Push current index
        stack.push(i);
    }

    return result;
};

```

# Implementation 2: Reverse Traversal with Stack

```typescript
/**
 * Definition for singly-linked list.
 * class ListNode {
 * val: number
 * next: ListNode | null
 * constructor(val?: number, next?: ListNode | null) {
 * this.val = (val===undefined ? 0 : val)
 * this.next = (next===undefined ? null : next)
 * }
 * }
 */

function nextLargerNodes(head: ListNode | null): number[] {
    // Convert to array first
    const values: number[] = [];
    let current = head;
    while (current !== null) {
        values.push(current.val);
        current = current.next;
    }

    const n = values.length;
    const result = new Array(n).fill(0);

    // Stack to store values in decreasing order
    const stack: number[] = [];

    // Traverse from right to left
    for (let i = n - 1; i >= 0; i--) {
        // Remove elements smaller than or equal to current
        while (stack.length > 0 && stack[stack.length - 1] <= values[i]) {
            stack.pop();
        }

        // Top of stack is the next greater element
        if (stack.length > 0) {
            result[i] = stack[stack.length - 1];
        }

        // Push current value
        stack.push(values[i]);
    }

    return result;
};

```

# Example walkthrough

## Example 1:
**Input:** `head = [2,1,5]`

### Using Implementation 1 (Forward with indices):
```
values = [2, 1, 5]
result = [0, 0, 0]

i = 0: values[0] = 2
  stack is empty
  stack = [0]

i = 1: values[1] = 1
  values[0] = 2 > values[1] = 1, don't pop
  stack = [0, 1]

i = 2: values[2] = 5
  values[1] = 1 < values[2] = 5, pop index 1
    result[1] = 5
  values[0] = 2 < values[2] = 5, pop index 0
    result[0] = 5
  stack is empty
  stack = [2]

result = [5, 5, 0]
```

**Output:** `[5,5,0]`

## Example 2:
**Input:** `head = [2,7,4,3,5]`

### Using Implementation 1:
```
values = [2, 7, 4, 3, 5]
result = [0, 0, 0, 0, 0]

i = 0: values[0] = 2
  stack = [0]

i = 1: values[1] = 7
  values[0] = 2 < 7, pop 0
    result[0] = 7
  stack = [1]

i = 2: values[2] = 4
  values[1] = 7 > 4, don't pop
  stack = [1, 2]

i = 3: values[3] = 3
  values[2] = 4 > 3, don't pop
  stack = [1, 2, 3]

i = 4: values[4] = 5
  values[3] = 3 < 5, pop 3
    result[3] = 5
  values[2] = 4 < 5, pop 2
    result[2] = 5
  values[1] = 7 > 5, don't pop
  stack = [1, 4]

result = [7, 0, 5, 5, 0]
```

**Output:** `[7,0,5,5,0]`

# Comparison of Implementations

| Aspect | Impl 1 (Forward) | Impl 2 (Reverse) |
|--------|-----------------|-----------------|
| **Direction** | Left to right | Right to left |
| **Stack stores** | Indices | Values |
| **Readability** | Clearer logic | Slightly less intuitive |
| **Performance** | Same O(n) | Same O(n) |

---

# Key Points
- **Monotonic stack**: Maintains elements in decreasing order.
- **Pop smaller elements**: When we find a larger element, all smaller elements have found their next greater.
- **Array conversion**: Necessary because linked lists don't support random access.
- **Single pass**: Each element is pushed and popped at most once.
