[Q2. How Many Numbers Are Smaller Than the Current Number](https://leetcode.com/problems/how-many-numbers-are-smaller-than-the-current-number/)

# Intuition

Since the range of `nums[i]` is small (0 to 100), we can efficiently count the frequency of each number and then compute how many numbers are smaller than each element using prefix sums.

---

# Approach

1. Create a frequency array `freq` of size 101 (since `nums[i]` ranges from 0 to 100).
2. Populate `freq` with counts of each number in `nums`.
3. Compute a prefix sum array `prefix` where `prefix[i]` gives the total count of numbers less than `i`.
4. For each element in `nums`, the count of smaller numbers is `prefix[nums[i]]`.
5. Return the array of these counts.

---

# Complexity

- **Time complexity:**  $$O(n + 101)$$, where `n` is the length of `nums`. The counting and prefix sum computation are linear, and the final iteration is linear.
  
- **Space complexity:**  $$O(101)$$ for the frequency and prefix arrays, which is constant.

---

# Code

```typescript
function smallerNumbersThanCurrent(nums: number[]): number[] {
    const freq = new Array(101).fill(0);
    for (const num of nums) {
        freq[num]++;
    }

    const prefix = new Array(101).fill(0);
    for (let i = 1; i <= 100; i++) {
        prefix[i] = prefix[i - 1] + freq[i - 1];
    }

    const result = nums.map(num => prefix[num]);
    return result;
};

```

# Example walkthrough
## Example 1:

Input: `[8,1,2,2,3]`

- freq: counts of each number
- prefix: cumulative counts of numbers less than each index
- For each number:
  - 8 → prefix[8]= count of numbers less than 8 → 4
  - 1 → prefix[1]= 0
  - 2 → prefix[2]= 1
  - 2 → prefix[2]= 1
  - 3 → prefix[3]= 2

Output: `[4,0,1,1,3]`

This approach is efficient and leverages the small range of input values for quick computation.
