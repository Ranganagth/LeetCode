# Intuition

This problem is a classic "Largest Rectangle in Histogram" problem. The key idea is to use a stack to keep track of bars in increasing height order:
- When a bar is lower than the top of the stack, it means we've found a boundary for the rectangle with the height of the bar at the top of the stack.
- We pop from the stack and calculate the area with the popped bar as the smallest height.
- We continue this process until the stack is empty or the current bar is taller than the top of the stack.
- The maximum area found during this process is the answer.

This approach ensures each bar is pushed and popped at most once, leading to an efficient solution.

---

# Approach

1. Initialize an empty stack to store indices of bars.
2. Append a zero height bar at the end of `heights` to flush out remaining bars.
3. Iterate through each bar:
   - While the current bar height is less than the height of the bar at the top of the stack:
     - Pop the top index.
     - Calculate the area with the popped bar as the smallest height:
       - Width = current index - index of new top of stack - 1 (if stack not empty), else current index.
     - Update max area if needed.
   - Push current index onto the stack.
4. Return the maximum area found.

---

# Complexity

- **Time complexity:** $$O(n)$$, since each bar is pushed and popped at most once.
  
- **Space complexity:**  $$O(n)$$ for the stack.

---

# Code

```typescript
function largestRectangleArea(heights: number[]): number {
    const stack: number[] = [];
    let maxArea = 0;
    heights.push(0); // Append a sentinel to flush remaining bars

    for (let i = 0; i < heights.length; i++) {
        while (stack.length > 0 && heights[i] < heights[stack[stack.length - 1]]) {
            const height = heights[stack.pop()!];
            const width = stack.length === 0 ? i : i - stack[stack.length - 1] - 1;
            maxArea = Math.max(maxArea, height * width);
        }
        stack.push(i);
    }

    return maxArea;
};

```

# Example walkthrough
## Example 1:

Input: `[2,1,5,6,2,3]`

- Append 0: `[2,1,5,6,2,3,0]`
- i=0: stack=[0]
- i=1: 1 < 2 → pop 0, area=2*1=2; stack=[]
  - push 1 → stack=[1]
- i=2: 5 > 1 → push 2 → stack=[1,2]
- i=3: 6 > 5 → push 3 → stack=[1,2,3]
- i=4: 2 < 6 → pop 3, area=6*(4-2-1)=6*1=6
  - 2 < 5 → pop 2, area=5*(4-1-1)=5*2=10
  - 2 ≥ 1 → push 4 → stack=[1,4]
- i=5: 3 > 2 → push 5 → stack=[1,4,5]
- i=6: 0 < 3 → pop 5, area=3*(6-4-1)=3*1=3
  - 0 < 2 → pop 4, area=2*(6-1-1)=2*4=8
  - 0 < 1 → pop 1, area=1*(6-0-1)=1*5=5
  - push 6 → stack=[6]

Max area found: 10

This method efficiently computes the largest rectangle area in the histogram.
