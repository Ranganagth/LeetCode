[Q2. Evaluate Reverse Polish Notation](https://leetcode.com/problems/evaluate-reverse-polish-notation/)

# Intuition

Reverse Polish Notation (RPN) can be efficiently evaluated using a stack:
- Push operands onto the stack.
- When encountering an operator, pop the top two operands, perform the operation, and push the result back.
- Continue until all tokens are processed.
- The final value on the stack is the result.

---

# Approach

1. Initialize an empty stack.
2. Iterate through each token:
   - If it's a number, convert to integer and push onto the stack.
   - If it's an operator, pop the top two numbers, perform the operation, and push the result.
3. After processing all tokens, the stack contains one element, which is the result.

---

# Code

```typescript
function evalRPN(tokens: string[]): number {
    const stack: number[] = [];

    for (const token of tokens) {
        if (!isNaN(Number(token))) {
            // Token is a number
            stack.push(Number(token));
        } else {
            // Token is an operator
            const b = stack.pop()!;
            const a = stack.pop()!;
            let result: number;

            switch (token) {
                case '+':
                    result = a + b;
                    break;
                case '-':
                    result = a - b;
                    break;
                case '*':
                    result = a * b;
                    break;
                case '/':
                    // Truncate toward zero
                    result = Math.trunc(a / b);
                    break;
                default:
                    throw new Error(`Invalid operator: ${token}`);
            }
            stack.push(result);
        }
    }

    return stack.pop()!;
};

```

# Example walkthrough
## Example 1:

Input: `["2","1","+","3","*"]`

- Push 2
- Push 1
- '+' → pop 1 and 2, compute 2+1=3, push 3
- Push 3
- '*' → pop 3 and 3, compute 3*3=9, push 9
- Final stack: [9]
- Output: 9

This approach is efficient, straightforward, and handles all constraints.
