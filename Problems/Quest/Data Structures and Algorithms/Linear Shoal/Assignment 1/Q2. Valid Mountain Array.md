# Intuition

A valid mountain array must have:
1. **Length >= 3**
2. A **strictly increasing** part from the start
3. A **peak** (not at the start or end)
4. A **strictly decreasing** part after the peak

We can use a **two-pointer approach**:
- Start from the left and count how many consecutive strictly increasing elements.
- Start from the right and count how many consecutive strictly decreasing elements.
- Check if both pointers meet at a valid peak position (not at start or end).

---

# Approach

1. Check if `arr.length < 3` → return false.
2. Initialize `left = 0` and `right = arr.length - 1`.
3. **Move left pointer right** while `arr[left] < arr[left + 1]` (strictly increasing).
4. **Move right pointer left** while `arr[right] < arr[right - 1]` (strictly decreasing).
5. Check if:
   - `left > 0` (at least one element in the increasing part)
   - `right < arr.length - 1` (at least one element in the decreasing part)
   - `left == right` (they meet at the peak)

---

# Complexity

- **Time complexity:** 
  $$O(n)$$, where `n` is the length of the array. We traverse the array at most twice (once from left, once from right).

- **Space complexity:** 
  $$O(1)$$ for constant extra space.

---

# Code

```typescript
function validMountainArray(arr: number[]): boolean {
    // Mountain array must have at least 3 elements
    if (arr.length < 3) return false;

    let left = 0;
    let right = arr.length - 1;

    // Move left pointer while strictly increasing
    while (left + 1 < arr.length && arr[left] < arr[left + 1]) {
        left++;
    }

    // Move right pointer while strictly decreasing
    while (right > 0 && arr[right] < arr[right - 1]) {
        right--;
    }

    // Check if peak is valid (not at start or end) and both pointers meet
    return left > 0 && right < arr.length - 1 && left === right;
};

```

# Example walkthrough
## Example 1:
**Input:** `arr = [2,1]`

- Length = 2 < 3 → return false

**Output:** `false`

## Example 2:
**Input:** `arr = [3,5,5]`

- left = 0: arr[0] = 3 < arr[1] = 5 → left = 1
- left = 1: arr[1] = 5 NOT < arr[2] = 5 → stop, left = 1
- right = 2: arr[2] = 5 NOT < arr[1] = 5 → stop, right = 2
- Check: left = 1 > 0, right = 2 NOT < 2
- return false

**Output:** `false`

## Example 3:
**Input:** `arr = [0,3,2,1]`

- left = 0: arr[0] = 0 < arr[1] = 3 → left = 1
- left = 1: arr[1] = 3 NOT < arr[2] = 2 → stop, left = 1
- right = 3: arr[3] = 1 < arr[2] = 2 → right = 2
- right = 2: arr[2] = 2 NOT < arr[1] = 3... wait, 2 < 3 → right = 1
- right = 1: arr[1] = 3 NOT < arr[0] = 0 → stop, right = 1
- Check: left = 1 > 0, right = 1 < 3, left = 1 === right = 1
- return true

**Output:** `true`

## Example 4 (Edge case):
**Input:** `arr = [0,1,2,3,4,5,4,3,2,1]`

- left moves to index 5 (peak)
- right moves to index 5 (peak)
- Check: left = 5 > 0, right = 5 < 9, left === right
- return true

**Output:** `true`

## Example 5 (Edge case):
**Input:** `arr = [1,2,3,4,5]`

- left moves to index 4 (end)
- right stays at index 4
- Check: left = 4 > 0 , right = 4 NOT < 4 
- return false (no decreasing part)

**Output:** `false`

This two-pointer approach is efficient and handles all edge cases correctly.
