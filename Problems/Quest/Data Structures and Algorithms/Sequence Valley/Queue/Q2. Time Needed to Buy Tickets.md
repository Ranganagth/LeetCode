[Q2. Time Needed to Buy Tickets](https://leetcode.com/problems/time-needed-to-buy-tickets/)

# Intuition

The problem involves simulating a queue where each person buys one ticket at a time and moves to the back of the line until they finish buying all their tickets. To determine the time taken for the person at position `k` to finish buying tickets, we need to:
1. Track how many tickets each person has left to buy.
2. Simulate the process of buying tickets until the person at position `k` finishes.

Instead of simulating the entire queue, we can calculate the time directly by observing:
- Each person contributes to the total time based on the number of tickets they need to buy.
- For the person at position `k`, we only need to count the time until they finish buying all their tickets.

---

# Approach

1. Initialize a variable `time` to track the total time taken.
2. Loop through the array `tickets`:
   - For each person `i`:
     - If `i <= k`, they contribute `min(tickets[i], tickets[k])` seconds to the total time (since they will buy tickets until the person at `k` finishes).
     - If `i > k`, they contribute `min(tickets[i], tickets[k] - 1)` seconds (since they will stop buying tickets once the person at `k` finishes).
3. Return the total time.

---

# Complexity

- **Time complexity:** 
  $$O(n)$$, where `n` is the length of the `tickets` array, as we iterate through the array once.

- **Space complexity:** 
  $$O(1)$$, as we use only a few variables.

---

# Code

```typescript
function timeRequiredToBuy(tickets: number[], k: number): number {
    let time = 0;

    for (let i = 0; i < tickets.length; i++) {
        if (i <= k) {
            time += Math.min(tickets[i], tickets[k]);
        } else {
            time += Math.min(tickets[i], tickets[k] - 1);
        }
    }

    return time;
};

```

# Example walkthrough

## Example 1:
**Input:** `tickets = [2,3,2], k = 2`

**Process:**
- Person 0: `min(2, 2) = 2` seconds
- Person 1: `min(3, 2) = 2` seconds
- Person 2: `min(2, 2) = 2` seconds

**Total time:** `2 + 2 + 2 = 6`

**Output:** `6`

---

## Example 2:
**Input:** `tickets = [5,1,1,1], k = 0`

**Process:**
- Person 0: `min(5, 5) = 5` seconds
- Person 1: `min(1, 5) = 1` second
- Person 2: `min(1, 5) = 1` second
- Person 3: `min(1, 5) = 1` second

**Total time:** `5 + 1 + 1 + 1 = 8`

**Output:** `8`

This approach efficiently calculates the time required for the person at position `k` to finish buying tickets without simulating the entire queue.
