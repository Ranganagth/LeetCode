[Q3. Implement Queue using Stacks](https://leetcode.com/problems/implement-queue-using-stacks/)

# Intuition

To implement a queue using two stacks, we need to simulate the **FIFO (First-In-First-Out)** behavior of a queue using the **LIFO (Last-In-First-Out)** behavior of stacks. The key idea is to use two stacks:
- **Input stack:** Used to push new elements.
- **Output stack:** Used to pop and peek elements in the correct order.

When popping or peeking, if the output stack is empty, we transfer all elements from the input stack to the output stack. This ensures the oldest element (front of the queue) is at the top of the output stack.

---

# Approach

1. **Push operation:** Push the element directly onto the input stack.
2. **Pop operation:** 
   - If the output stack is empty, transfer all elements from the input stack to the output stack.
   - Pop the top element from the output stack.
3. **Peek operation:** 
   - If the output stack is empty, transfer all elements from the input stack to the output stack.
   - Return the top element of the output stack.
4. **Empty operation:** Check if both stacks are empty.

---

# Complexity

- **Time complexity:**
  - `push`: $$O(1)$$
  - `pop`: $$O(1)$$ amortized (since transferring elements happens only when the output stack is empty)
  - `peek`: $$O(1)$$ amortized
  - `empty`: $$O(1)$$
- **Space complexity:**$$O(n)$$, where `n` is the number of elements in the queue (stored across two stacks).

# Code

```typescript
class MyQueue {
    private inputStack: number[];
    private outputStack: number[];

    constructor() {
        this.inputStack = [];
        this.outputStack = [];
    }

    push(x: number): void {
        this.inputStack.push(x);
    }

    pop(): number {
        if (this.outputStack.length === 0) {
            while (this.inputStack.length > 0) {
                this.outputStack.push(this.inputStack.pop()!);
            }
        }
        return this.outputStack.pop()!;
    }

    peek(): number {
        if (this.outputStack.length === 0) {
            while (this.inputStack.length > 0) {
                this.outputStack.push(this.inputStack.pop()!);
            }
        }
        return this.outputStack[this.outputStack.length - 1];
    }

    empty(): boolean {
        return this.inputStack.length === 0 && this.outputStack.length === 0;
    }
}

 * var param_3 = obj.peek()
 * var param_4 = obj.empty()
 */
```

# Example walkthrough
## Example 1:
**Input:** 
`["MyQueue", "push", "push", "peek", "pop", "empty"]` 
`[[], [1], [2], [], [], []]`

**Process:**
1. `MyQueue myQueue = new MyQueue();` 
   - Initialize `inputStack = []`, `outputStack = []`.
2. `myQueue.push(1);` 
   - Push 1 onto `inputStack`: `inputStack = [1]`.
3. `myQueue.push(2);` 
   - Push 2 onto `inputStack`: `inputStack = [1, 2]`.
4. `myQueue.peek();` 
   - `outputStack` is empty, transfer elements from `inputStack`: 
     - Pop 2 → Push to `outputStack`: `outputStack = [2]`.
     - Pop 1 → Push to `outputStack`: `outputStack = [2, 1]`. 
   - Peek top of `outputStack`: `1`.
5. `myQueue.pop();` 
   - Pop top of `outputStack`: `outputStack = [2]`. 
   - Return `1`.
6. `myQueue.empty();` 
   - Check both stacks: `inputStack = []`, `outputStack = [2]`. 
   - Return `false`.

**Output:** `[null, null, null, 1, 1, false]`

---

This implementation ensures efficient queue operations using two stacks with amortized $O(1)$ time complexity for `pop` and `peek`.
