[Q1. Number of Students Unable to Eat Lunch](https://leetcode.com/problems/number-of-students-unable-to-eat-lunch/description/?envType=problem-list-v2&envId=dsa-sequence-valley-queue)

# Intuition

The key insight is that students go to the back of the queue if they don't want the top sandwich. A student can only eat if their preferred sandwich type is still available when they reach the front. The process stops when we cycle through all remaining students without anyone taking a sandwich.

The correct approach is to count how many students prefer each type and how many sandwiches of each type exist. A student can eat only if there's a matching sandwich available. We need to match students with sandwiches in order, and any unmatched students cannot eat.

---

# Approach

1. Count the number of students preferring each type (0 and 1).
2. Iterate through the sandwiches in order:
   - If there are students preferring this sandwich type, one student eats it (decrement the count).
   - If there are no students preferring this type, the sandwich cannot be eaten.
3. After processing all sandwiches, the remaining students who prefer a type but have no matching sandwiches left cannot eat.
4. Return the count of students unable to eat.

---

# Complexity

- **Time complexity:** 
  $$O(n)$$, where `n` is the length of students/sandwiches arrays.

- **Space complexity:** 
  $$O(1)$$, as we only use a fixed-size count array.

---

# Code

```typescript
function countStudents(students: number[], sandwiches: number[]): number {
    let countPref = [0, 0]; // countPref[0] for students preferring 0, countPref[1] for 1

    for (const s of students) {
        countPref[s]++;
    }

    // Process each sandwich in order
    for (const sandwich of sandwiches) {
        if (countPref[sandwich] > 0) {
            // A student who prefers this sandwich takes it
            countPref[sandwich]--;
        } else {
            // No student wants this sandwich, process stops here
            // All remaining students cannot eat
            return countPref[0] + countPref[1];
        }
    }

    // All sandwiches were taken
    return 0;
};

```

# Example walkthrough

## Example 1:
Input: `students = [1,1,0,0], sandwiches = [0,1,0,1]`

**Process/Approach explanation:**
- countPref: [2 (preferring 0), 2 (preferring 1)]
- Sandwich 0: countPref[0]=2 > 0 → take it, countPref=[1,2]
- Sandwich 1: countPref[1]=2 > 0 → take it, countPref=[1,1]
- Sandwich 0: countPref[0]=1 > 0 → take it, countPref=[0,1]
- Sandwich 1: countPref[1]=1 > 0 → take it, countPref=[0,0]
- All sandwiches taken

**Output:** `0`

## Example 2:
Input: `students = [1,1,1,0,0,1], sandwiches = [1,0,0,0,1,1]`

**Process/Approach explanation:**
- countPref: [2 (preferring 0), 4 (preferring 1)]
- Sandwich 1: countPref[1]=4 > 0 → take it, countPref=[2,3]
- Sandwich 0: countPref[0]=2 > 0 → take it, countPref=[1,3]
- Sandwich 0: countPref[0]=1 > 0 → take it, countPref=[0,3]
- Sandwich 0: countPref[0]=0 → no student wants it, stop
- Remaining students: countPref[0]+countPref[1]=0+3=3

**Output:** `3`

This approach correctly identifies when the process stops and counts the students unable to eat.
