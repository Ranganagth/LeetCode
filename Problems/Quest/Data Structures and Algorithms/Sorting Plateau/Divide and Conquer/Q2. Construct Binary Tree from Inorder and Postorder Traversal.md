# Intuition

The key insight is understanding the properties of inorder and postorder traversals:

- **Postorder:** Left -> Right -> Root (last element is always the root)
- **Inorder:** Left -> Root -> Right (root divides left and right subtrees)

We can recursively build the tree by:
1. Taking the last element from postorder as the root
2. Finding that root in inorder to split left and right subtrees
3. Recursively building left and right subtrees

---

# Approach

1. **Base case:** If arrays are empty, return null

2. **Find root:** Last element in postorder is the root

3. **Find root position in inorder:** This divides the array into left and right subtrees

4. **Recursively build:**
   - Left subtree: elements before root in inorder
   - Right subtree: elements after root in inorder
   - Use corresponding portions of postorder

5. **Optimization:** Use a hash map to quickly find root positions in inorder

---

# Complexity

- **Time complexity:** O(n) - visit each node once, O(1) lookup with hash map

- **Space complexity:** O(n) - hash map storage + recursion stack (O(h) where h is height)

# Code

## Solution 1: Recursive with Hash Map (Optimal)

```typescript
function buildTree(inorder: number[], postorder: number[]): TreeNode | null {
    // Create hash map for O(1) lookup of indices in inorder
    const inorderMap = new Map<number, number>();
    for (let i = 0; i < inorder.length; i++) {
        inorderMap.set(inorder[i], i);
    }
    
    // Index to track current root in postorder (from right to left)
    let postIndex = postorder.length - 1;
    
    function build(inStart: number, inEnd: number): TreeNode | null {
        // Base case: no elements to construct tree
        if (inStart > inEnd) {
            return null;
        }
        
        // Get root value from postorder (from right to left)
        const rootVal = postorder[postIndex--];
        const root = new TreeNode(rootVal);
        
        // Find root position in inorder
        const inIndex = inorderMap.get(rootVal)!;
        
        // Build right subtree first (because we're going right to left in postorder)
        root.right = build(inIndex + 1, inEnd);
        
        // Build left subtree
        root.left = build(inStart, inIndex - 1);
        
        return root;
    }
    
    return build(0, inorder.length - 1);
};

```

## Solution 2: With Array Slicing (Less Efficient but Clearer)

```typescript
function buildTree(inorder: number[], postorder: number[]): TreeNode | null {
    // Base case
    if (inorder.length === 0 || postorder.length === 0) {
        return null;
    }
    
    // Last element in postorder is the root
    const rootVal = postorder[postorder.length - 1];
    const root = new TreeNode(rootVal);
    
    // Find root position in inorder
    const rootIndex = inorder.indexOf(rootVal);
    
    // Split inorder into left and right subtrees
    const leftInorder = inorder.slice(0, rootIndex);
    const rightInorder = inorder.slice(rootIndex + 1);
    
    // Split postorder accordingly
    // Left subtree has same number of elements as leftInorder
    const leftPostorder = postorder.slice(0, rootIndex);
    const rightPostorder = postorder.slice(rootIndex, postorder.length - 1);
    
    // Recursively build subtrees
    root.left = buildTree(leftInorder, leftPostorder);
    root.right = buildTree(rightInorder, rightPostorder);
    
    return root;
};

```

## Solution 3: Iterative with Stack

```typescript
function buildTree(inorder: number[], postorder: number[]): TreeNode | null {
    if (postorder.length === 0) return null;
    
    const root = new TreeNode(postorder[postorder.length - 1]);
    const stack: TreeNode[] = [root];
    let inorderIndex = inorder.length - 1;
    
    // Process postorder from right to left (excluding last element)
    for (let i = postorder.length - 2; i >= 0; i--) {
        const val = postorder[i];
        let node = stack[stack.length - 1];
        
        // If current value is not in the right subtree
        if (node.val !== inorder[inorderIndex]) {
            // Add as right child
            node.right = new TreeNode(val);
            stack.push(node.right);
        } else {
            // Pop nodes until we find the parent
            while (stack.length > 0 && stack[stack.length - 1].val === inorder[inorderIndex]) {
                node = stack.pop()!;
                inorderIndex--;
            }
            // Add as left child
            node.left = new TreeNode(val);
            stack.push(node.left);
        }
    }
    
    return root;
};

```

## Solution 4: With Detailed Comments

```typescript
function buildTree(inorder: number[], postorder: number[]): TreeNode | null {
    // Map to store value -> index mapping for inorder array
    const inorderIndexMap = new Map<number, number>();
    inorder.forEach((val, idx) => inorderIndexMap.set(val, idx));
    
    // Pointer for postorder array (process from right to left)
    let postorderIndex = postorder.length - 1;
    
    function buildSubtree(left: number, right: number): TreeNode | null {
        // Base case: invalid range
        if (left > right) {
            return null;
        }
        
        // Current root is the last unprocessed element in postorder
        const rootValue = postorder[postorderIndex];
        postorderIndex--;
        
        // Create root node
        const root = new TreeNode(rootValue);
        
        // Find where this root divides the inorder array
        const rootIndexInInorder = inorderIndexMap.get(rootValue)!;
        
        // Important: Build RIGHT subtree first
        // Because postorder is Left-Right-Root, when going backwards it's Root-Right-Left
        root.right = buildSubtree(rootIndexInInorder + 1, right);
        
        // Then build LEFT subtree
        root.left = buildSubtree(left, rootIndexInInorder - 1);
        
        return root;
    }
    
    return buildSubtree(0, inorder.length - 1);
};

```

# Example Walkthrough

## Example 1:

**Input:** `inorder = [9,3,15,20,7], postorder = [9,15,7,20,3]`

**Step-by-step construction:**

```
Step 1: Root
- postorder last element: 3
- Find 3 in inorder at index 1
- Left subtree: [9], Right subtree: [15,20,7]

     3
    / \
   ? ?

Step 2: Build right subtree of 3
- postorder: [9,15,7,20] -> last is 20
- Find 20 in inorder at index 3
- Left of 20: [15], Right of 20: [7]

     3
    / \
   ? 20
      / \
     ? ?

Step 3: Build right child of 20
- postorder: [9,15,7] -> last is 7
- Find 7 in inorder at index 4
- No children

     3
    / \
   ? 20
      / \
     ? 7

Step 4: Build left child of 20
- postorder: [9,15] -> last is 15
- Find 15 in inorder at index 2
- No children

     3
    / \
   ? 20
      / \
    15 7

Step 5: Build left child of 3
- postorder: [9] -> last is 9
- Find 9 in inorder at index 0
- No children

     3
    / \
   9 20
      / \
    15 7
```

**Output:** `[3,9,20,null,null,15,7]`

```
                            L R L R R

Key insight:
- Last element in postorder is always root
- Root in inorder divides left and right subtrees
```

---

## Key Insights

- **Postorder processing:** Go from right to left (last element is root)
- **Build order:** Build RIGHT subtree before LEFT (important!)
- **Hash map optimization:** O(1) lookup instead of O(n) search
- **Index tracking:** Use indices instead of array slicing for efficiency
- **Recursion pattern:** Each recursive call handles a subtree

---

## Why Build Right First?

**Postorder is Left-Right-Root:**
```
When processing backwards: Root-Right-Left

Example: [9, 15, 7, 20, 3]
Backwards: 3 (root), 20 (right), 7 (right-right), 15 (right-left), 9 (left)

So we encounter right subtree nodes before left subtree nodes
```

## Edge Cases

| Case | Input | Output | Notes |
|------|-------|--------|-------|
| Single node | inorder=[1], postorder=[1] | [1] | Minimal tree |
| Left skewed | inorder=[3,2,1], postorder=[3,2,1] | Left chain | No right children |
| Right skewed | inorder=[1,2,3], postorder=[1,2,3] | Right chain | No left children |
| Complete tree | inorder=[4,2,5,1,6,3,7], postorder=[4,5,2,6,7,3,1] | Complete binary tree | Balanced |

---

## Common Mistakes to Avoid

1. **Building left before right:** Must build right first when processing postorder backwards
2. **Not using hash map:** O(n) search for each node makes it O(n^2)
3. **Wrong array slicing:** Off-by-one errors in calculating subtree sizes
4. **Forgetting to decrement postIndex:** Must move pointer after using each element

---

## Performance Comparison

| Approach | Time | Space | Notes |
|----------|------|-------|-------|
| With hash map | O(n) | O(n) | Optimal |
| With indexOf | O(n^2) | O(n) | Slow for large trees |
| Array slicing | O(n^2) | O(n^2) | Creates many arrays |

**Use Solution 1 (Hash Map)** for:
- Optimal O(n) time complexity
- Clean and efficient code
- Best for interviews and production
- Easy to explain and understand

This is the standard and most efficient solution for this problem.
