# Intuition

This problem is similar to counting inversions but with a twist: we need `nums[i] > 2 * nums[j]` instead of just `nums[i] > nums[j]`.

The brute force O(n^2) approach is too slow for n = 50,000. We need an O(n log n) solution using **merge sort** with a modification to count reverse pairs during the merge process.

**Key insight:** During merge sort, when the array is split and both halves are sorted, we can efficiently count pairs where the left element is more than twice the right element.

---

# Approach

## Solution: Modified Merge Sort

1. **Divide:** Split array into two halves recursively

2. **Count pairs:** Before merging, count reverse pairs between left and right halves
   - For each element in left half, find how many elements in right half satisfy the condition
   - Use two pointers since both halves are sorted

3. **Merge:** Merge the two sorted halves (standard merge sort)

4. **Return:** Total count from all recursive calls

**Why this works:**
- Pairs within left half are counted in left recursion
- Pairs within right half are counted in right recursion
- Pairs across halves are counted before merging
- Sorting enables efficient counting with two pointers

---

# Complexity

- **Time complexity:** O(n log n) - merge sort with linear counting at each level

- **Space complexity:** O(n) - temporary arrays for merging

# Code

## Solution 1: Merge Sort with Reverse Pair Counting

```typescript
function reversePairs(nums: number[]): number {
    return mergeSort(nums, 0, nums.length - 1);
    
    function mergeSort(arr: number[], left: number, right: number): number {
        if (left >= right) {
            return 0;
        }
        
        const mid = Math.floor((left + right) / 2);
        
        // Count pairs in left and right halves
        let count = mergeSort(arr, left, mid) + mergeSort(arr, mid + 1, right);
        
        // Count pairs across the two halves
        count += countPairs(arr, left, mid, right);
        
        // Merge the two sorted halves
        merge(arr, left, mid, right);
        
        return count;
    }
    
    function countPairs(arr: number[], left: number, mid: number, right: number): number {
        let count = 0;
        let j = mid + 1;
        
        // For each element in left half
        for (let i = left; i <= mid; i++) {
            // Find how many elements in right half satisfy nums[i] > 2 * nums[j]
            while (j <= right && arr[i] > 2 * arr[j]) {
                j++;
            }
            count += j - (mid + 1);
        }
        
        return count;
    }
    
    function merge(arr: number[], left: number, mid: number, right: number): void {
        const temp: number[] = [];
        let i = left;
        let j = mid + 1;
        
        // Merge two sorted halves
        while (i <= mid && j <= right) {
            if (arr[i] <= arr[j]) {
                temp.push(arr[i++]);
            } else {
                temp.push(arr[j++]);
            }
        }
        
        // Copy remaining elements
        while (i <= mid) {
            temp.push(arr[i++]);
        }
        while (j <= right) {
            temp.push(arr[j++]);
        }
        
        // Copy back to original array
        for (let k = 0; k < temp.length; k++) {
            arr[left + k] = temp[k];
        }
    }
};

```

## Solution 2: With Detailed Comments

```typescript
function reversePairs(nums: number[]): number {
    return mergeSortAndCount(nums, 0, nums.length - 1);
    
    function mergeSortAndCount(arr: number[], start: number, end: number): number {
        // Base case: single element or empty
        if (start >= end) {
            return 0;
        }
        
        const mid = Math.floor((start + end) / 2);
        
        // Recursively count in left half
        let count = mergeSortAndCount(arr, start, mid);
        
        // Recursively count in right half
        count += mergeSortAndCount(arr, mid + 1, end);
        
        // Count pairs across the two halves
        // This is the key step where we count reverse pairs
        count += countReversePairs(arr, start, mid, end);
        
        // Merge the sorted halves
        mergeArrays(arr, start, mid, end);
        
        return count;
    }
    
    function countReversePairs(arr: number[], start: number, mid: number, end: number): number {
        let count = 0;
        let rightPointer = mid + 1;
        
        // For each element in the left sorted half
        for (let leftPointer = start; leftPointer <= mid; leftPointer++) {
            // Move right pointer to find all elements where arr[left] > 2 * arr[right]
            // Since both halves are sorted, we can use two pointers efficiently
            while (rightPointer <= end && arr[leftPointer] > 2 * arr[rightPointer]) {
                rightPointer++;
            }
            
            // All elements from (mid+1) to (rightPointer-1) form valid pairs
            count += rightPointer - (mid + 1);
        }
        
        return count;
    }
    
    function mergeArrays(arr: number[], start: number, mid: number, end: number): void {
        const merged: number[] = [];
        let left = start;
        let right = mid + 1;
        
        // Standard merge process
        while (left <= mid && right <= end) {
            if (arr[left] <= arr[right]) {
                merged.push(arr[left++]);
            } else {
                merged.push(arr[right++]);
            }
        }
        
        // Copy remaining elements from left half
        while (left <= mid) {
            merged.push(arr[left++]);
        }
        
        // Copy remaining elements from right half
        while (right <= end) {
            merged.push(arr[right++]);
        }
        
        // Write merged array back to original
        for (let i = 0; i < merged.length; i++) {
            arr[start + i] = merged[i];
        }
    }
};
```

## Solution 3: Using Binary Indexed Tree (Alternative)

```typescript
function reversePairs(nums: number[]): number {
    // Coordinate compression
    const sorted = [...new Set(nums.map(x => x).concat(nums.map(x => 2 * x)))].sort((a, b) => a - b);
    const rank = new Map<number, number>();
    sorted.forEach((val, idx) => rank.set(val, idx + 1));
    
    const n = sorted.length;
    const bit = new Array(n + 1).fill(0);
    
    function update(idx: number): void {
        while (idx <= n) {
            bit[idx]++;
            idx += idx & -idx;
        }
    }
    
    function query(idx: number): number {
        let sum = 0;
        while (idx > 0) {
            sum += bit[idx];
            idx -= idx & -idx;
        }
        return sum;
    }
    
    let count = 0;
    
    // Process from right to left
    for (let i = nums.length - 1; i >= 0; i--) {
        // Count how many numbers are less than nums[i] / 2
        const target = rank.get(nums[i])!;
        count += query(target - 1);
        
        // Add 2 * nums[i] to the tree
        const doubleRank = rank.get(2 * nums[i])!;
        update(doubleRank);
    }
    
    return count;
};

```

# Example Walkthrough

## Example 1:

**Input:** `nums = [1,3,2,3,1]`

**Merge sort process:**

```
Initial: [1, 3, 2, 3, 1]

Split:
[1, 3, 2] | [3, 1]

Further split:
[1, 3] | [2] | [3] | [1]

[1] | [3] | [2] | [3] | [1]

Merge and count:
[1, 3]: count = 0 (no reverse pairs)
[3, 1]: count = 1 (3 > 2*1)

[1, 2, 3]: 
- Count pairs between [1,3] and [2]: 0
- Merge: [1, 2, 3]

[1, 3]:
- Count pairs between [3] and [1]: 1 (3 > 2*1)
- Merge: [1, 3]

Final merge [1, 2, 3] and [1, 3]:
- Count pairs: 1 (3 > 2*1 from position 1 and 4)
- Total: 0 + 1 + 0 + 1 = 2
```

**Pairs found:**
- (1, 4): nums[1] = 3, nums[4] = 1, 3 > 2*1
- (3, 4): nums[3] = 3, nums[4] = 1, 3 > 2*1

**Output:** `2`

## Key Insights

- **Two-pointer technique:** Both halves are sorted, so we can efficiently count pairs
- **Separate counting and merging:** Count before merging to avoid interference
- **Overflow prevention:** Be careful with `2 * nums[j]` for large values
- **Order matters:** Count pairs where left element > 2 * right element

---

## Why Merge Sort?

**Advantages:**
1. Divides problem into smaller subproblems
2. Sorted halves enable efficient counting
3. O(n log n) time complexity
4. Natural fit for this type of problem

**Comparison with brute force:**
- Brute force: O(n^2) - too slow
- Merge sort: O(n log n) - optimal

## Edge Cases

| Case | Input | Output | Notes |
|------|-------|--------|-------|
| Single element | [1] | 0 | No pairs |
| All ascending | [1,2,3,4] | 0 | No reverse pairs |
| All descending | [4,3,2,1] | Multiple | Many pairs |
| With negatives | [-1,-2,3] | Varies | Handle carefully |
| Large values | [2^31-1, 1] | 1 | Overflow risk |

---

## Common Mistakes

1. **Counting during merge:** Must count before merging
2. **Integer overflow:** `2 * nums[j]` can overflow
3. **Wrong pointer logic:** Must handle sorted halves correctly
4. **Not resetting pointers:** Each left element needs fresh right pointer
