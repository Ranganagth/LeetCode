# Intuition

The key insight is understanding the constraint: for any `i < k < j`, we need `2 * nums[k] != nums[i] + nums[j]`.

This means `nums[k]` should never be the average of `nums[i]` and `nums[j]`.

**Mathematical insight:**
- If we separate odd and even numbers, an odd number can never be the average of two numbers (since the sum of any two numbers is either both odd or both even, making the average either an integer or a half-integer)
- We can recursively construct beautiful arrays by:
  1. Building a beautiful array for smaller n
  2. Transforming it: all odd positions on left, all even positions on right
  3. Using the property: `2 * odd != odd + even` and `2 * even != odd + even`

---

# Approach

## Solution: Divide and Conquer

The elegant solution uses a recursive approach:

1. **Base case:** For n = 1, return [1]

2. **Recursive construction:**
   - Build a beautiful array for n
   - Transform by mapping: `x -> 2*x - 1` (creates odd numbers)
   - Transform by mapping: `x -> 2*x` (creates even numbers)
   - Concatenate: odds + evens

3. **Why this works:**
   - Odd numbers on left, even numbers on right
   - No element between an odd and even can be their average
   - Within odds/evens, the recursive structure maintains the property

---

# Complexity

- **Time complexity:** O(n log n) - recursive construction with merging

- **Space complexity:** O(n log n) - recursion stack and intermediate arrays

# Code

## Solution 1: Divide and Conquer (Elegant)

```typescript
function beautifulArray(n: number): number[] {
    // Base case
    if (n === 1) return [1];
    
    // Recursively build beautiful array
    const result: number[] = [];
    
    // Get beautiful array for smaller size
    const smaller = beautifulArray(Math.floor((n + 1) / 2));
    
    // Add odd numbers (2*x - 1)
    for (const x of smaller) {
        const odd = 2 * x - 1;
        if (odd <= n) {
            result.push(odd);
        }
    }
    
    // Add even numbers (2*x)
    const smaller2 = beautifulArray(Math.floor(n / 2));
    for (const x of smaller2) {
        const even = 2 * x;
        if (even <= n) {
            result.push(even);
        }
    }
    
    return result;
};

```

## Solution 2: Iterative Construction

```typescript
function beautifulArray(n: number): number[] {
    let result = [1];
    
    while (result.length < n) {
        const temp: number[] = [];
        
        // Add odd transformations
        for (const x of result) {
            const odd = 2 * x - 1;
            if (odd <= n) {
                temp.push(odd);
            }
        }
        
        // Add even transformations
        for (const x of result) {
            const even = 2 * x;
            if (even <= n) {
                temp.push(even);
            }
        }
        
        result = temp;
    }
    
    return result;
};

```

## Solution 3: Simplified Recursive

```typescript
function beautifulArray(n: number): number[] {
    if (n === 1) return [1];
    
    const odds: number[] = [];
    const evens: number[] = [];
    
    // Recursively get beautiful array
    const prev = beautifulArray(n);
    
    // Separate into odds and evens
    for (const num of prev) {
        if (num % 2 === 1) {
            odds.push(num);
        } else {
            evens.push(num);
        }
    }
    
    return [...odds, ...evens];
};

```

## Solution 4: Optimized with Memoization

```typescript
function beautifulArray(n: number): number[] {
    const memo = new Map<number, number[]>();
    
    function build(size: number): number[] {
        if (size === 1) return [1];
        if (memo.has(size)) return memo.get(size)!;
        
        const result: number[] = [];
        
        // Build odds
        const oddSize = Math.floor((size + 1) / 2);
        const odds = build(oddSize);
        for (const x of odds) {
            result.push(2 * x - 1);
        }
        
        // Build evens
        const evenSize = Math.floor(size / 2);
        const evens = build(evenSize);
        for (const x of evens) {
            result.push(2 * x);
        }
        
        memo.set(size, result);
        return result;
    }
    
    return build(n);
};

```

# Example Walkthrough

## Example 1:

**Input:** `n = 4`

**Construction:**

```
n = 4

Step 1: Build for n = 2
  - Build for n = 1: [1]
  - Odds: 2*1-1 = 1 -> [1]
  - Evens: 2*1 = 2 -> [2]
  - Result: [1, 2]

Step 2: Build for n = 4 using [1, 2]
  - Odds from [1,2]: 2*1-1=1, 2*2-1=3 -> [1, 3]
  - Evens from [1,2]: 2*1=2, 2*2=4 -> [2, 4]
  - Result: [1, 3, 2, 4]
```

**Verification:**
```
Array: [1, 3, 2, 4]
Check all triplets (i, k, j):
- (0,1,2): 2*3 != 1+2 (6 != 3)
- (0,1,3): 2*3 != 1+4 (6 != 5)
- (0,2,3): 2*2 != 1+4 (4 != 5)
- (1,2,3): 2*2 != 3+4 (4 != 7)
```

**Output:** `[1,3,2,4]` or `[2,1,4,3]` (multiple valid answers)

## Key Insights

- **Odd-Even separation:** The core insight that prevents averaging
- **Recursive structure:** Each level maintains the beautiful property
- **Linear transformation:** `2*x - 1` and `2*x` preserve the property
- **Multiple solutions:** Many valid beautiful arrays exist for each n
- **No need to verify:** The construction guarantees correctness

---

## Why This Works

**Mathematical proof:**

1. **Base case:** [1] is trivially beautiful

2. **Inductive step:** If A is beautiful, then:
   - Odds = {2*a - 1 : a in A} are all odd
   - Evens = {2*a : a in A} are all even
   - For any i < k < j:
     - If i, j both in odds or both in evens: property holds by induction
     - If i in odds, j in evens: 2*nums[k] can't equal odd + even (wrong parity)
     - If i in evens, j in odds: impossible since odds come first
