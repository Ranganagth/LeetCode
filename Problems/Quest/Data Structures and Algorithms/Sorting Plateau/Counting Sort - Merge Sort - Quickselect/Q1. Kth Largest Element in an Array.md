# Intuition

The problem asks for the kth largest element. While sorting is straightforward, we can solve this more efficiently using:
1. **QuickSelect algorithm** - O(n) average time
2. **Min Heap** - O(n log k) time
3. **Max Heap** - O(n + k log n) time

The QuickSelect approach is optimal for average case performance.

---

# Approach

## Solution 1: QuickSelect (Optimal - O(n) average)

QuickSelect is similar to QuickSort but only recurses into one partition:
1. Choose a pivot element
2. Partition array so elements >= pivot are on left, elements < pivot are on right
3. If pivot is at position k-1 (0-indexed), we found the answer
4. Otherwise, recurse into the appropriate partition

## Solution 2: Min Heap of Size K

Maintain a min heap of size k containing the k largest elements seen so far:
1. Add first k elements to heap
2. For remaining elements, if element > heap top, remove top and add element
3. The top of heap is the kth largest

## Solution 3: Sorting (Simple but O(n log n))

Sort in descending order and return element at index k-1.

---

# Complexity

**QuickSelect:**
- **Time complexity:** O(n) average, O(n^2) worst case
- **Space complexity:** O(log n) for recursion stack

**Min Heap:**
- **Time complexity:** O(n log k)
- **Space complexity:** O(k)

**Sorting:**
- **Time complexity:** O(n log n)
- **Space complexity:** O(1) or O(n) depending on sort algorithm

# Code

## Solution 1: QuickSelect (Optimal)

```typescript
function findKthLargest(nums: number[], k: number): number {
    // QuickSelect to find kth largest (which is (n-k)th smallest in 0-indexed)
    const n = nums.length;
    return quickSelect(nums, 0, n - 1, n - k);
    
    function quickSelect(arr: number[], left: number, right: number, kSmallest: number): number {
        if (left === right) {
            return arr[left];
        }
        
        // Partition and get pivot index
        const pivotIndex = partition(arr, left, right);
        
        if (kSmallest === pivotIndex) {
            return arr[kSmallest];
        } else if (kSmallest < pivotIndex) {
            return quickSelect(arr, left, pivotIndex - 1, kSmallest);
        } else {
            return quickSelect(arr, pivotIndex + 1, right, kSmallest);
        }
    }
    
    function partition(arr: number[], left: number, right: number): number {
        // Choose random pivot for better average performance
        const randomIndex = left + Math.floor(Math.random() * (right - left + 1));
        [arr[randomIndex], arr[right]] = [arr[right], arr[randomIndex]];
        
        const pivot = arr[right];
        let i = left;
        
        for (let j = left; j < right; j++) {
            if (arr[j] <= pivot) {
                [arr[i], arr[j]] = [arr[j], arr[i]];
                i++;
            }
        }
        
        [arr[i], arr[right]] = [arr[right], arr[i]];
        return i;
    }
};

```

## Solution 2: Min Heap Approach

```typescript
function findKthLargest(nums: number[], k: number): number {
    // Min heap to store k largest elements
    const minHeap: number[] = [];
    
    // Helper functions for min heap
    const parent = (i: number) => Math.floor((i - 1) / 2);
    const leftChild = (i: number) => 2 * i + 1;
    const rightChild = (i: number) => 2 * i + 2;
    
    const swap = (i: number, j: number) => {
        [minHeap[i], minHeap[j]] = [minHeap[j], minHeap[i]];
    };
    
    const heapifyUp = (index: number) => {
        while (index > 0 && minHeap[parent(index)] > minHeap[index]) {
            swap(index, parent(index));
            index = parent(index);
        }
    };
    
    const heapifyDown = (index: number) => {
        let minIndex = index;
        const left = leftChild(index);
        const right = rightChild(index);
        
        if (left < minHeap.length && minHeap[left] < minHeap[minIndex]) {
            minIndex = left;
        }
        if (right < minHeap.length && minHeap[right] < minHeap[minIndex]) {
            minIndex = right;
        }
        
        if (minIndex !== index) {
            swap(index, minIndex);
            heapifyDown(minIndex);
        }
    };
    
    const push = (val: number) => {
        minHeap.push(val);
        heapifyUp(minHeap.length - 1);
    };
    
    const pop = (): number => {
        if (minHeap.length === 0) return -1;
        const min = minHeap[0];
        minHeap[0] = minHeap[minHeap.length - 1];
        minHeap.pop();
        if (minHeap.length > 0) {
            heapifyDown(0);
        }
        return min;
    };
    
    // Build heap with first k elements
    for (let i = 0; i < k; i++) {
        push(nums[i]);
    }
    
    // Process remaining elements
    for (let i = k; i < nums.length; i++) {
        if (nums[i] > minHeap[0]) {
            pop();
            push(nums[i]);
        }
    }
    
    return minHeap[0];
};

```

## Solution 3: Simple Sorting

```typescript
function findKthLargest(nums: number[], k: number): number {
    // Sort in descending order
    nums.sort((a, b) => b - a);
    
    // Return kth largest (k-1 index since 0-indexed)
    return nums[k - 1];
};

```

## Solution 4: Using Built-in Priority Queue (if available)

```typescript
function findKthLargest(nums: number[], k: number): number {
    // Sort and return kth largest
    return nums.sort((a, b) => b - a)[k - 1];
};
```

# Example Walkthrough

## Example 1:

**Input:** `nums = [3,2,1,5,6,4], k = 2`

**QuickSelect Approach:**

1. We want 2nd largest, which is (6-2)=4th smallest (0-indexed: index 4)
2. Choose pivot, partition array
3. Suppose after partition: [3,2,1,4,5,6] with pivot at index 3
4. We want index 4, so recurse right
5. Continue until we find element at correct position

**Sorted view:** [6, 5, 4, 3, 2, 1]
**2nd largest:** 5

**Output:** `5`

---

## Example 2:

**Input:** `nums = [3,2,3,1,2,4,5,5,6], k = 4`

**Min Heap Approach:**

1. Build min heap with first 4 elements: [1, 2, 2, 3]
2. Process remaining: 3, 4, 5, 5, 6
   - 3 > 1: remove 1, add 3 -> [2, 2, 3, 3]
   - 4 > 2: remove 2, add 4 -> [2, 3, 3, 4]
   - 5 > 2: remove 2, add 5 -> [3, 3, 4, 5]
   - 5 > 3: remove 3, add 5 -> [3, 4, 5, 5]
   - 6 > 3: remove 3, add 6 -> [4, 5, 5, 6]
3. Min of heap is 4

**Sorted view:** [6, 5, 5, 4, 3, 3, 2, 2, 1]
**4th largest:** 4

**Output:** `4`

## Key Insights

- **QuickSelect:** Best average case O(n), but can degrade to O(n^2) with bad pivots
- **Random pivot:** Helps avoid worst case in QuickSelect
- **Min Heap:** Consistent O(n log k) performance, good when k is small
- **Kth largest = (n-k)th smallest:** Important conversion for QuickSelect
- **In-place:** QuickSelect modifies array but uses O(1) extra space

---

## Performance Comparison

| Approach | Time (Avg) | Time (Worst) | Space | Best For |
|----------|------------|--------------|-------|----------|
| QuickSelect | O(n) | O(n^2) | O(log n) | General case |
| Min Heap | O(n log k) | O(n log k) | O(k) | Small k |
| Sorting | O(n log n) | O(n log n) | O(1) | Simple solution |

**Recommended:** QuickSelect for optimal average performance, Min Heap for guaranteed O(n log k).
