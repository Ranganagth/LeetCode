# Intuition

The array is a rotated sorted array, meaning it was originally sorted but then rotated at some pivot point. The key insight is that despite the rotation, at least one half of the array (left or right of mid) will always be properly sorted.

We can use modified binary search by:
1. Identifying which half is sorted
2. Checking if the target lies in the sorted half
3. Searching the appropriate half

---

# Approach

1. **Initialize pointers:** `left = 0`, `right = nums.length - 1`

2. **Binary search loop:** While `left <= right`:
   - Calculate `mid`
   - If `nums[mid] == target`, return `mid`
   - **Determine which half is sorted:**
     - If `nums[left] <= nums[mid]`: left half is sorted
       - If target is in range `[nums[left], nums[mid])`, search left
       - Otherwise, search right
     - Else: right half is sorted
       - If target is in range `(nums[mid], nums[right]]`, search right
       - Otherwise, search left

3. **Not found:** Return `-1`

---

# Complexity

- **Time complexity:** O(log n) - binary search with constant time decisions

- **Space complexity:** O(1) - only using pointers

# Code Analysis

```typescript
function search(nums: number[], target: number): number {
    let left = 0, right = nums.length - 1;

    while (left <= right) {
        const mid = Math.floor((left + right) / 2);

        // Found target
        if (nums[mid] === target) {
            return mid;
        }

        // Check if left half is sorted
        if (nums[left] <= nums[mid]) {
            // Left half [left...mid] is sorted
            // Check if target is in this sorted range
            if (target >= nums[left] && target < nums[mid]) {
                // Target is in left sorted half
                right = mid - 1;
            } else {
                // Target is in right half (possibly rotated)
                left = mid + 1;
            }
        } else {
            // Right half [mid...right] is sorted
            // Check if target is in this sorted range
            if (target > nums[mid] && target <= nums[right]) {
                // Target is in right sorted half
                left = mid + 1;
            } else {
                // Target is in left half (possibly rotated)
                right = mid - 1;
            }
        }
    }

    return -1;
};

```

## Solution 2: Find Pivot First

```typescript
function search(nums: number[], target: number): number {
    const n = nums.length;
    
    // Find the pivot (smallest element)
    let left = 0, right = n - 1;
    while (left < right) {
        const mid = Math.floor((left + right) / 2);
        if (nums[mid] > nums[right]) {
            left = mid + 1;
        } else {
            right = mid;
        }
    }
    
    const pivot = left;
    
    // Determine which half to search
    left = 0;
    right = n - 1;
    
    if (target >= nums[pivot] && target <= nums[right]) {
        left = pivot;
    } else {
        right = pivot - 1;
    }
    
    // Standard binary search
    while (left <= right) {
        const mid = Math.floor((left + right) / 2);
        if (nums[mid] === target) {
            return mid;
        } else if (nums[mid] < target) {
            left = mid + 1;
        } else {
            right = mid - 1;
        }
    }
    
    return -1;
};

```

## Solution 3: With Detailed Comments

```typescript
function search(nums: number[], target: number): number {
    let left = 0;
    let right = nums.length - 1;

    while (left <= right) {
        const mid = Math.floor((left + right) / 2);

        if (nums[mid] === target) {
            return mid;
        }

        // Determine which side is properly sorted
        const leftSorted = nums[left] <= nums[mid];
        
        if (leftSorted) {
            // Left side is sorted: nums[left] ... nums[mid]
            const targetInLeftRange = target >= nums[left] && target < nums[mid];
            
            if (targetInLeftRange) {
                right = mid - 1; // Search left
            } else {
                left = mid + 1; // Search right
            }
        } else {
            // Right side is sorted: nums[mid] ... nums[right]
            const targetInRightRange = target > nums[mid] && target <= nums[right];
            
            if (targetInRightRange) {
                left = mid + 1; // Search right
            } else {
                right = mid - 1; // Search left
            }
        }
    }

    return -1;
};

```

## Key Insights

- **At least one half is always sorted:** This is guaranteed by the rotation property
- **Comparison with endpoints:** `nums[left] <= nums[mid]` determines which half is sorted
- **Range checking:** Once we know which half is sorted, we check if target is in that range
- **Binary search maintained:** We always eliminate half the search space

---

## Why This Works

**Case 1: Left half sorted (nums[left] <= nums[mid])**
```
[4, 5, 6, 7, 0, 1, 2]
 L M R

Left [4,5,6,7] is sorted
If target in [4,7), search left
Otherwise, search right (which contains the rotation)
```

**Case 2: Right half sorted (nums[left] > nums[mid])**
```
[6, 7, 0, 1, 2, 4, 5]
 L M R

Right [1,2,4,5] is sorted
If target in (1,5], search right
Otherwise, search left (which contains the rotation)
```

## Common Mistakes to Avoid

1. **Wrong sorted half check:** Must use `<=` not `<` in `nums[left] <= nums[mid]`
2. **Incorrect range bounds:** Use `<` for left range, `<=` for right range
3. **Missing edge cases:** Handle single element and no rotation cases
4. **Off-by-one errors:** Be careful with `mid - 1` and `mid + 1`
