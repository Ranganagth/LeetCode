# Intuition

A mountain array has a single peak where values increase up to that peak and then decrease. We need to find the peak index in O(log n) time, which suggests using binary search.

The key insight is that at any position:
- If `arr[mid] < arr[mid + 1]`, we're on the ascending slope, so the peak is to the right
- If `arr[mid] > arr[mid + 1]`, we're on the descending slope or at the peak, so the peak is at mid or to the left

---

# Approach

Use binary search to find the peak:

1. **Initialize pointers:** `left = 0`, `right = arr.length - 1`

2. **Binary search:** While `left < right`:
   - Calculate `mid = left + (right - left) / 2`
   - Compare `arr[mid]` with `arr[mid + 1]`
   - If ascending (`arr[mid] < arr[mid + 1]`), move left pointer: `left = mid + 1`
   - If descending (`arr[mid] > arr[mid + 1]`), move right pointer: `right = mid`

3. **Return result:** When `left == right`, we've found the peak

**Why this works:**
- We're always moving toward the peak
- The peak is the only position where `arr[i-1] < arr[i] > arr[i+1]`
- Binary search guarantees O(log n) time

---

# Complexity

- **Time complexity:** O(log n) - binary search halves the search space each iteration

- **Space complexity:** O(1) - only using a few pointers

# Code

## Solution 1: Standard Binary Search

```typescript
function peakIndexInMountainArray(arr: number[]): number {
    let left = 0;
    let right = arr.length - 1;
    
    while (left < right) {
        const mid = left + Math.floor((right - left) / 2);
        
        if (arr[mid] < arr[mid + 1]) {
            // We're on ascending slope, peak is to the right
            left = mid + 1;
        } else {
            // We're on descending slope or at peak, peak is at mid or left
            right = mid;
        }
    }
    
    // When left == right, we've found the peak
    return left;
};

```

```typescript
function peakIndexInMountainArray(arr: number[]): number {
    let left = 0;
    let right = arr.length - 1;
    
    while (left < right) {
        const mid = left + Math.floor((right - left) / 2);
        
        // Check if we're on the ascending or descending part
        if (arr[mid] < arr[mid + 1]) {
            // Ascending: arr[mid] < arr[mid+1]
            // Peak must be to the right (including mid+1)
            left = mid + 1;
        } else {
            // Descending: arr[mid] > arr[mid+1]
            // Peak is at mid or to the left
            right = mid;
        }
    }
    
    // left == right at the peak
    return left;
};

```

## Solution 4: Using Three-Way Comparison

```typescript
function peakIndexInMountainArray(arr: number[]): number {
    let left = 1; // Start from 1 since peak can't be at index 0
    let right = arr.length - 2; // End at length-2 since peak can't be at last index
    
    while (left <= right) {
        const mid = Math.floor((left + right) / 2);
        
        if (arr[mid - 1] < arr[mid] && arr[mid] > arr[mid + 1]) {
            // Found the peak
            return mid;
        } else if (arr[mid - 1] < arr[mid]) {
            // Ascending, go right
            left = mid + 1;
        } else {
            // Descending, go left
            right = mid - 1;
        }
    }
    
    return left;
};

```

## Key Insights

- **Binary search pattern:** Similar to finding a target, but we're searching for a property (peak)
- **Invariant:** The peak is always within [left, right]
- **Comparison direction:** We only need to compare with one neighbor (mid+1)
- **No equality check:** We never check if mid is the peak explicitly in the main solution
- **Convergence:** left and right converge to the peak index

---

## Why This Works

**Proof of correctness:**

1. **Initial state:** Peak is somewhere in [0, n-1]

2. **Loop invariant:** Peak is always in [left, right]

3. **Progress:** Each iteration halves the search space

4. **Termination:** When left == right, we've found the peak

5. **Why no explicit peak check needed:**
   - If ascending, peak must be to the right
   - If descending, peak is at mid or to the left
   - Eventually, left and right converge to the peak

## Common Mistakes to Avoid

1. **Using `left <= right`:** Should be `left < right` to avoid infinite loop
2. **Wrong mid calculation:** Use `left + (right - left) / 2` to avoid overflow
3. **Comparing with wrong neighbor:** Always compare with `mid + 1`
4. **Not updating correctly:** `left = mid + 1` vs `right = mid` is crucial

## Performance Analysis

For array of length n:
- **Iterations:** log2(n)
- **Operations per iteration:** O(1)
- **Total:** O(log n)

Example: n = 1000
- Linear search: 1000 comparisons worst case
- Binary search: 10 comparisons worst case
