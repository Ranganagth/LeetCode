# Intuition

This is a classic binary search problem. Since the array is sorted in ascending order and we need O(log n) complexity, binary search is the perfect approach.

The key idea is to repeatedly divide the search space in half by comparing the target with the middle element, eliminating half of the remaining elements each time.

---

# Approach

1. **Initialize pointers:** `left = 0`, `right = nums.length - 1`

2. **Binary search loop:** While `left <= right`:
   - Calculate middle index: `mid = left + (right - left) / 2`
   - If `nums[mid] == target`, return `mid`
   - If `nums[mid] < target`, search right half: `left = mid + 1`
   - If `nums[mid] > target`, search left half: `right = mid - 1`

3. **Target not found:** If loop ends without finding target, return `-1`

---

# Complexity

- **Time complexity:** O(log n) - search space halves each iteration

- **Space complexity:** O(1) - only using a few pointers

# Code

## Solution 1: Standard Binary Search

```typescript
function search(nums: number[], target: number): number {
    let left = 0;
    let right = nums.length - 1;
    
    while (left <= right) {
        const mid = left + Math.floor((right - left) / 2);
        
        if (nums[mid] === target) {
            return mid;
        } else if (nums[mid] < target) {
            // Target is in right half
            left = mid + 1;
        } else {
            // Target is in left half
            right = mid - 1;
        }
    }
    
    // Target not found
    return -1;
};

```

## Solution 3: Recursive Approach

```typescript
function search(nums: number[], target: number): number {
    return binarySearch(nums, target, 0, nums.length - 1);
    
    function binarySearch(arr: number[], target: number, left: number, right: number): number {
        if (left > right) {
            return -1;
        }
        
        const mid = left + Math.floor((right - left) / 2);
        
        if (arr[mid] === target) {
            return mid;
        } else if (arr[mid] < target) {
            return binarySearch(arr, target, mid + 1, right);
        } else {
            return binarySearch(arr, target, left, mid - 1);
        }
    }
};

```

function search(nums: number[], target: number): number {
    // Early return for edge cases
    if (nums.length === 0) return -1;
    if (target < nums[0] || target > nums[nums.length - 1]) return -1;
    
    let left = 0;
    let right = nums.length - 1;

        } else {

        }

## Key Insights

- **Mid calculation:** Use `left + (right - left) / 2` to avoid integer overflow
- **Loop condition:** `left <= right` allows checking single element
- **Update rules:** 
  - `left = mid + 1` when target is larger
  - `right = mid - 1` when target is smaller
- **Termination:** Loop ends when `left > right`, meaning target not found

---

## Common Mistakes to Avoid

1. **Wrong mid calculation:** `(left + right) / 2` can overflow for large values
   - **Correct:** `left + (right - left) / 2`

2. **Wrong loop condition:** Using `left < right` instead of `left <= right`
   - Misses the case when there's only one element left

3. **Wrong update:** Using `left = mid` or `right = mid`
   - Can cause infinite loops

4. **Forgetting floor:** In TypeScript, use `Math.floor()` for integer division

## When to Use Binary Search

**Requirements:**
- Array must be sorted
- Need O(log n) time complexity
- Random access available (arrays, not linked lists)

**Variations:**
- Find first/last occurrence
- Find insertion position
- Search in rotated sorted array
- Find peak element
