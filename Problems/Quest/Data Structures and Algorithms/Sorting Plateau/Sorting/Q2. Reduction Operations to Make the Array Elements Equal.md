# Intuition

The key is to think about **how many elements need to be reduced at each distinct value level**.

When we sort the array and find a new distinct larger value at position `i`, all elements **before position `i`** (which are smaller) will eventually need to be reduced. Since there are `N - i` elements from position `i` onwards that are greater than or equal to the current value, each of those elements will trigger one operation for the smaller elements.

---

# Approach

1. **Sort the array** - This groups equal values together and allows us to process values in ascending order.

2. **Iterate through sorted array** - For each position `i` from 1 to N-1, check if we've encountered a new distinct larger value.

3. **Count operations** - When `nums[i-1] < nums[i]`, we've found a new distinct value. All elements from index `i` onwards (there are `N - i` of them) will need to be reduced to this new level. So we add `N - i` to our operation count.

4. **Why this works** - Each element at a higher level needs to be reduced once for every distinct value below it. By counting from the perspective of "how many elements are at or above this level", we efficiently count all operations.

---

# Complexity

- **Time complexity:** $$O(n \log n)$$ - dominated by sorting the array

- **Space complexity:** $$O(1)$$ - only using a few variables (excluding sorting space)

---

# Code

```typescript
function reductionOperations(nums: number[]): number {
    // Sort the array in ascending order
    nums.sort((a, b) => a - b);

    let ops = 0;
    const N = nums.length;
    
    // Iterate through the sorted array
    for (let i = 1; i < N; ++i) {
        // If we found a new distinct larger value
        if (nums[i - 1] < nums[i]) {
            // All elements from index i onwards need one operation
            // to reduce to the previous distinct value
            ops += N - i;
        }
    }

    return ops;
};

```

# Example Walkthrough

## Example 1:

**Input:** `nums = [5,1,3]`

**Explanation:**
1. Sort: `[1, 3, 5]` (N = 3)
2. Iterate:
   - i=1: `nums[0]=1 < nums[1]=3` → ops += (3 - 1) = 2 → ops = 2
     - Elements at indices 1,2 (values 3,5) need to reduce
   - i=2: `nums[1]=3 < nums[2]=5` → ops += (3 - 2) = 1 → ops = 3
     - Element at index 2 (value 5) needs to reduce

**Output:** `3`

---

## Example 2:

**Input:** `nums = [1,1,1]`

**Explanation:**
1. Sort: `[1, 1, 1]` (N = 3)
2. Iterate:
   - i=1: `nums[0]=1 < nums[1]=1` → no change
   - i=2: `nums[1]=1 < nums[2]=1` → no change
3. All elements are equal, no operations needed

**Output:** `0`

---

## Example 3:

**Input:** `nums = [1,1,2,2,3]`

**Explanation:**
1. Sort: `[1, 1, 2, 2, 3]` (N = 5)
2. Iterate:
   - i=1: `nums[0]=1 < nums[1]=1` → ops = 0
   - i=2: `nums[1]=1 < nums[2]=2` → ops += (5 - 2) = 3 → ops = 3
     - Elements at indices 2,3,4 (values 2,2,3) need to reduce to 1
   - i=3: `nums[2]=2 < nums[3]=2` → ops = 3
   - i=4: `nums[3]=2 < nums[4]=3` → ops += (5 - 4) = 1 → ops = 4
     - Element at index 4 (value 3) needs to reduce to 2

**Output:** `4`

---

## 💡 Key Insight

The formula `N - i` represents **how many elements are at or above the current distinct value level**. Each of these elements will perform exactly one operation when transitioning from the current level to the next lower distinct level.
